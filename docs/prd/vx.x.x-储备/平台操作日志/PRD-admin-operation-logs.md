## 1. 功能概述

### 1.1 功能定位

操作日志是管理后台（平台管理员视角）下的一个功能页面，定位为**运营平台操作审计中心**。它把分散在各业务模块（页面维度记录事件）的关键写操作，统一汇聚为一条条可检索、可下钻、可导出的审计事件，回答"运营平台上发生过什么、谁干的、成没成"。

近期已将原来的"多分类 Tab"简化为单一的操作日志，聚焦运营平台操作这一条主线，避免用户在多个日志分类间来回切换。

**范围边界**：
- 本页面仅记录与回溯**运营平台上的管理操作**（页面维度记录事件），用于安全审计与合规留痕。
- **不承担线上故障排查、用户投诉处理的职责**；页面也不提供面向排障的链路追踪检索能力。
- 页面 UI 不再单独展示"平台操作日志"标题文案，直接以检索条 + 列表呈现。
- 检索维度定为**操作人 + 时间范围**两项；操作模块下拉本期不做（见附录C-Q07）。

### 1.2 目标用户

| 用户类型 | 特征描述 | 核心诉求 |
|---------|---------|---------|
| 平台管理员（platform_admin） | 拥有平台最高权限，负责平台日常运营管理 | 快速回溯任意一次运营操作、定位异常操作人、导出审计证据 |
| 安全/合规审计人员 | 不定期或按需核查运营操作链路 | 按操作人/时间范围拉取完整操作流水并导出 |

### 1.3 核心价值

| 维度 | 描述 |
|------|------|
| 用户价值 | 让分散在各业务模块的运营操作有统一入口可查，一次筛选 + 一次展开即可还原操作全貌 |
| 业务价值 | 满足等保/内审对"运营操作可追溯、责任可界定"的合规底线 |
| 平台价值 | 统一运营平台操作的事实口径，避免各业务模块各记各的、口径不一 |

### 2.3 详细规格

#### F-01：日志列表展示

**功能描述**: 以表格形式按时间倒序展示审计事件，每行摘要呈现一次操作的核心要素。

**前置条件**:
- 用户已以 platform_admin 身份登录管理后台
- 后端审计服务正常、有审计事件写入

**主要行为**:
1. 进入"系统 → 日志"页面，自动加载最近 200 条审计事件
2. 加载中展示"加载中..."；无数据展示"暂无日志"
3. 表格列：时间 / 动作 / 对象 / 操作者 / 结果
4. 结果列用绿色"成功"或红色"失败"标签区分；无结果信息时显示"—"

**业务规则**:
- 列表按事件发生时间倒序排列（最新在最上）
- "动作"列展示操作的中文名称（如 `employee.create` → "创建员工"）；未配置中文映射的显示原始事件类型
- "对象"列优先展示操作对象的结构化信息（类型 / 名称 / ID），其次从业务载荷兜底（如"任务 xxx / 员工 xxx"）
- "操作者"列展示用户名（不展示角色，角色信息在详情中查看）

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 审计事件为空 | 表格区显示"暂无日志" |
| 操作者信息缺失 | 显示"—" |
| 对象信息缺失 | 显示"—" |
| 结果信息缺失 | 结果列显示"—" |
| 加载失败 | 列表置空（静默降级，建议增加提示，见附录C-Q01） |

---

#### F-02：检索（操作人 + 时间范围）

**功能描述**: 在列表上方提供检索条件，仅支持"操作人"与"时间范围"两个维度，二者可组合。检索条件改变后点击搜索刷新列表。

**前置条件**:
- 日志列表已加载

**主要行为**:
1. 提供**两个**检索项：操作人（搜索框）、时间范围（起止日期控件）
2. 点击"搜索"按钮触发按当前条件重新加载；起止日期变化即生效
3. 列表返回检索后的结果，统计卡同步更新

**业务规则**:
- **操作人**：文本搜索框，按用户名模糊匹配
- **时间范围**：起止两个日期选择器（天级精度）；结束日含当天全天（前端补到 23:59:59，避免漏结束日数据）
- 两个条件同时存在时取交集；均为空时返回最近 200 条
- **不提供**以下检索能力（本期明确不做）：关键词模糊、操作级别、具体操作事件类型、操作模块下拉、租户/实例

---

#### F-03：详情下钻

**功能描述**: 点击任意一行，在该行下方内联展开详情面板，呈现这次操作的完整 5W1H 与关联信息。

**前置条件**:
- 列表已加载且至少有一条记录

**主要行为**:
1. 点击行，下方展开详情面板；再次点击收起
2. 详情按"5W1H 审计要素"组织：操作者 / 时间 / 来源（IP + 设备）/ 对象 / 动作 / 结果
3. 详情含"关联信息"区：事件类型 / 事件ID / 请求ID / 链路ID（页面客观展示，作为后台记录核对依据）
4. 详情底部可展开"原始 JSON"，展示该事件的完整原始数据

**业务规则**:
- 同一时刻只展开一行（展开新行自动收起旧行）
- 结果区展示成功/失败及失败原因
- 来源区展示 IP 与 User-Agent
- 事件ID、请求ID、链路ID 为该事件的后台关联标识，便于与后台记录核对（不作为面向用户的检索维度）

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 某些 5W1H 字段缺失 | 对应字段显示"—" |
| 原始数据为空 | 原始 JSON 区显示空对象 |

---

#### F-04：数据导出

**功能描述**: 将当前检索结果导出为 CSV 或 JSON 文件，供留档或离线分析。

**前置条件**:
- 列表已有数据（或已应用检索）

**主要行为**:
1. 点击"CSV"或"JSON"按钮
2. 浏览器下载对应格式文件

**业务规则**:
- 导出范围 = 当前检索条件下的结果集（**不应**仅限当前页可见的 200 条，见附录C-Q03）
- CSV 导出固定字段：时间、范围、动作、操作者、状态（依据现状导出列）
- JSON 导出为完整事件结构（含详情字段）

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 列表为空时导出 | 导出空文件（仅表头/空数组） |
| 数据量极大 | 需支持全量导出（异步或分批，见附录C-Q03） |

---

#### F-08：访问鉴权

**功能描述**: 控制本页面的可见性与数据范围。

**业务规则**:
- 被赋予管理员角色的成员登录页面可见，后续根据系统中配置的角色资源边界而定
- 非该角色用户不可见菜单、不可调用接口
- 对日志页自身的访问/查询行为，也纳入审计记录

```

│  [操作人 搜索框] [起───至] [搜索]    [CSV][JSON] [↻]     │  检索条 + 导出/刷新（无标题文案）
```

## 4. 交互流程

### 4.1 主流程：检索与查看日志

```
[起始] 管理员进入「系统 → 日志」
    ↓
[加载] 自动拉取最近 200 条审计事件，展示"加载中..."
    ↓
[展示] 表格按时间倒序展示，统计卡更新（总数/最新时间）
    ↓
[判断] 是否需要缩小范围？
    ↓ [是]                          ↓ [否]
[检索] 输入操作人 / 选起止日期       [查看] 直接浏览列表
    ↓ 点击搜索                        ↓
[刷新] 列表按检索刷新                 ↓
    ↓                                 ↓
[查看] 点击目标行 → 展开详情（5W1H + 关联信息 + 原始 JSON）
    ↓
[判断] 是否需要留档？
    ↓ [是]              ↓ [否]
[导出] CSV/JSON 下载    [结束]
    ↓
[结束]
```

**流程说明**:
1. 进入即加载，无需手动触发首屏
2. 检索条件修改后需点击"搜索"生效（手动刷新）
3. 详情采用行内展开，不跳页、不弹窗，保持列表上下文

### 4.2 异常分支

| 异常场景 | 触发条件 | 处理方式 | 用户提示 |
|---------|---------|---------|---------|
| 加载失败 | 接口异常/网络中断 | 列表置空，静默降级 | 无显式提示（建议增加，见附录C-Q01） |
| 数据为空 | 无审计事件/检索无匹配 | 展示空状态行 | "暂无日志" |
| 权限不足 | 非 platform_admin | 菜单不可见、接口拒绝 | （由登录与权限体系拦截） |
| 导出失败 | 浏览器拦截下载 | 不生成文件 | 建议补充失败提示（见附录C-Q01） |
| 单次超量 | 结果超过 200 条 | 仅展示前 200 条 | 建议提示"仅显示最近 N 条，请导出或缩小范围"（见附录C-Q04） |

### 4.3 状态机

本功能为只读查询，无可变业务对象状态机。唯一可交互状态为详情行的展开/收起：

```
[行-收起] --[点击]--> [行-展开]
[行-展开] --[再次点击/点击他行]--> [行-收起]
```

## 5. 接口定义（产品视角）

> 本章只描述产品层面的查询能力与返回信息，不含具体技术实现方案。

### 5.1 查询操作日志

**能力**: 按条件查询审计事件列表，仅支持"操作人 + 时间范围"检索，支持限量、分页。

**查询参数（产品语义）**:
| 参数 | 含义 | 是否必备（产品诉求） | 说明 |
|------|------|------|------|
| 操作人 | 按用户名过滤 | P0 | 模糊匹配 |
| 时间范围 | 起止时间 | P0 | 起止日期，结束日含当天全天 |
| 数量上限 | 返回条数 | P0 | 默认 200，需支持更大范围/分页 |
| 分页游标 | 翻页 | P1 | 突破单次上限（附录C-Q04） |

> 明确不提供：关键词、操作级别、具体操作事件类型、操作模块、租户/实例 等检索参数（本期不做）。

**返回信息（每条审计事件）**: 见第 6 章"审计事件字段"。

### 5.2 导出审计日志

**能力**: 按当前检索条件导出 CSV / JSON。

**产品规则**:
- 导出范围应 = 当前检索条件下的全量结果（不只是当前页 200 条）
- CSV 为精简字段，JSON 为完整事件结构
- 大数据量导出应支持全量（附录C-Q03）

## 6. 数据模型（产品视角：审计事件）

> 一条审计事件 = 一次被记录的运营平台操作。字段从产品语义角度定义。

### 6.1 审计事件字段说明

| 字段 | 产品含义 | 取值示例 | 是否必填 |
|------|---------|---------|---------|
| 事件ID | 本次审计记录的唯一标识 | `audit-xxxx` | 是 |
| 事件类型 | 操作的机器标识，用于分类（其前缀即"操作模块"） | `employee.create`、`login` | 是 |
| 动作 | 操作的人类可读动作（默认同事件类型） | 创建员工 / 登录 | 是 |
| 发生时间 | 操作发生时间（ISO 8601） | 2026-06-17T10:00:00Z | 是 |
| 操作者 | 执行操作的用户 | { 用户名, 角色 } | 可空（系统操作时） |
| 操作对象 | 被操作的资源 | { 对象类型, 名称, ID } | 可空 |
| 来源 | 操作发起的来源 | { IP, User-Agent } | 可空 |
| 操作结果 | 操作成败及失败原因 | { 是否成功, 失败原因 } | 可空 |
| 请求ID / 链路ID / 关联ID | 该事件的后台关联标识，用于与后台记录核对 | req-xxx / trace-xxx | 可空 |
| 上下文 | 附加业务上下文（租户、实例等） | { 租户ID, 实例ID, ... } | 可空 |
| 业务载荷 | 该操作特有的业务细节 | { employeeId, skillId, taskId } | 可空 |

### 6.2 事件类型清单（按业务域分类）

> 本期不提供"操作模块"下拉检索；下表业务域仅作为事件类型的分类参考与中文映射来源。
> 以下包含运管平台、管理后台两个平台的事件类型

| 操作模块（业务域） | 事件类型 |
|--------|---------|
| 身份认证 | 登录、登出 |
| 权限管理 | 创建/更新/删除用户，创建/更新角色 |
| 员工管理 | 创建/更新/删除员工、更新员工策略、启动/停止/重建实例 |
| 技能管理 | 创建/更新/删除技能、审批/驳回/发布技能、绑定/解绑技能 |
| 工具管理 | 创建/更新/审批工具 |
| 网关配置 | 添加/更新模型 |
| 租户管理 | 创建/暂停/激活租户 |

### 6.3 实体关系

```
[平台管理员] 1 ── N [审计事件]   （一个管理员产生多条操作）
[审计事件]   N ── 1 [操作对象]   （一条事件指向一个被操作资源）
```

## 7. 验收标准

### 7.1 功能验收

| 场景ID | 场景描述 | 操作步骤 | 预期结果 | 优先级 |
|--------|---------|---------|---------|--------|
| AC-01 | 首屏加载 | 以 platform_admin 登录 → 进入日志页 | 自动加载最近审计事件，表格倒序展示，统计卡显示总数/最新时间 | P0 |
| AC-02 | 展开详情 | 点击任一行 | 行下展开 5W1H + 关联信息 + 原始 JSON；再次点击收起 | P0 |
| AC-03 | 按操作人检索 | 在操作人框输入用户名 → 搜索 | 列表仅剩匹配该用户名的操作 | P0 |
| AC-04 | 按时间范围检索 | 选起止日期 → 搜索 | 列表仅剩该时段事件（含结束日当天） | P0 |
| AC-05 | 组合检索 | 同时输入操作人 + 选时间范围 → 搜索 | 列表为两者交集 | P0 |
| AC-06 | 导出 CSV | 点击 CSV | 下载 CSV，字段完整 | P0 |
| AC-07 | 导出 JSON | 点击 JSON | 下载 JSON，含完整事件结构 | P0 |
| AC-08 | 空状态 | 检索无匹配 | 显示"暂无日志" | P0 |
| AC-09 | 权限拦截 | 非 platform_admin 访问 | 不可见菜单/接口拒绝 | P0 |
| AC-10 | 结果标签 | 查看含成败的事件 | 成功绿色、失败红色，失败带原因 | P0 |
| AC-11 | 无标题文案 | 进入页面 | 检索条左侧不再显示"平台操作日志"静态标题 | P0 |

### 7.2 性能验收

| 指标 | 目标值 | 测试方法 |
|-----|--------|---------|
| 页面首屏加载 | <2s（P95） | 真实数据量下手动/工具测 |
| 列表查询响应 | <500ms（P99，常规检索） | 接口压测 |
| 单次最大可加载 | ≥200 条无卡顿 | 大数据量回归 |

### 7.3 安全验收

| 检查项 | 验收标准 |
|--------|---------|
| 鉴权 | 未登录访问返回未授权 |
| 权限 | 非 platform_admin 不可访问、不可导出 |
| 数据隔离 | 仅可查看自身权限范围内的审计数据 |
| 留痕 | 对日志页的查询/导出行为本身也被审计 |
| 敏感信息 | 原始 JSON 中不出现明文密码等敏感字段 |

### 7.4 兼容性验收

| 环境 | 要求 |
|------|------|
| 浏览器 | Chrome 90+、Firefox 85+、Safari 14+、Edge 90+ |
| 屏幕分辨率 | 1280×720 及以上（桌面端为主） |
