## 1. 功能概述

### 1.1 功能定位

记忆库是 HMR Agent 管理平台的核心模块之一，负责存储、组织和检索 Agent 对用户的认知信息——包括用户画像（偏好与习惯）和工作规则（约束与规范）。它解决的核心问题是：Agent 每次对话都需要用户重复说明偏好和规则的沟通浪费，以及 Agent 因记忆错误导致的信任损耗。

### 1.2 目标用户

| 用户类型 | 特征描述 | 核心诉求 |
|---------|---------|---------|
| Agent 搭档 | 日常使用 Agent 辅助工作，高频交互 | Agent 能自动记住偏好和规则，不用反复说明 |
| 团队管理者 | 管理多个 Agent 和团队成员 | 团队规则被 Agent 正确理解和执行 |
| 运维/安全人员 | 对生产环境有严格操作规范 | 关键约束不被 Agent 遗忘或误记 |

### 1.3 核心价值

| 维度 | 描述 |
|------|------|
| 用户价值 | 减少 80% 的重复偏好说明，Agent 首次交互即可"懂你" |
| 业务价值 | 提升 Agent 可信度，降低因记忆错误导致的操作事故 |
| 技术价值 | 统一 Agent 认知数据模型，为多 Agent 协作奠定基础 |

### 2.3 详细规格

#### F-01：记忆列表展示

**功能描述**: 记忆库首页以类型分组展示所有记忆，每组内按 lastUpdated 倒序排列。每条记忆行显示类型图标、标题、摘要、meta信息条，含关联定时任务的记忆额外显示任务 badge。

**前置条件**:
- 用户已登录并进入记忆库页面

**主要行为**:
1. 页面加载，获取当前用户所有记忆数据
2. 按类型分为三组：画像（✦）、规则（◎）、纠正（✧）
3. 每组内按 lastUpdated 时间倒序排列
4. 每条记忆行展示：类型图标 + 标题 + 摘要 + 任务 badge（如有）+ meta信息条（置信度圆点 + 置信度文字 · 来源 · 观察次数 · 更新时间）+ 展开箭头
5. hover 记忆行时显示编辑/删除操作按钮
6. 点击记忆行展开详情区域

**业务规则**:
- 无时间分组，记忆是状态而非事件
- 某类型下无记忆时，该分组不展示
- 更新/追加操作后，记忆因 lastUpdated 更新而自动排到该类型组最前
- 页面统计显示记忆总数
- meta 信息条中：观察次数（count）仅 observe 来源的记忆展示，declare/correct 来源时为空

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 记忆库为空 | 展示空状态引导页，提示添加第一条记忆 |
| 单条记忆内容超长 | 摘要截取前 30 字符，详情区域展示全文 |
| 同一类型下记忆数 > 50 | 正常渲染，后续迭代考虑分页或虚拟滚动 |

#### F-02：记忆 CRUD

**功能描述**: 支持用户手动新增、编辑、删除记忆。新增时需选择类型、填写标题和内容。编辑时保留原始来源和证据链。删除需二次确认。

**前置条件**:
- 用户已登录

**主要行为**:
1. **新增**：点击「＋ 添加记忆」按钮，弹出新增弹窗，选择类型（画像/规则）、填写标题和内容，点击添加
2. **编辑**：hover 记忆行出现编辑按钮，点击后弹窗预填现有内容，修改后保存
3. **删除**：hover 记忆行出现删除按钮，点击后弹出删除确认弹窗

**业务规则**:
- 类型枚举固定为：画像（pref）、规则（know）；纠正类型由系统自动生成，用户不可主动选择
- 标题为必填，最大 50 字符
- 内容为必填，最大 2000 字符
- 新增记忆默认置信度为「中确」，来源为「声明」
- 编辑记忆时 lastUpdated 更新为当前时间，但来源（source）保持不变
- 新增/编辑时，若内容中检测到时间模式，弹窗底部展示提示条

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 标题或内容为空提交 | Toast 提示"请填写标题和内容"，不关闭弹窗 |
| 编辑时未做任何修改就保存 | 正常关闭弹窗，不触发更新逻辑 |
| 删除含关联定时任务的记忆 | 确认弹窗额外提示"关联的定时任务也将一并解除关联" |

---

#### F-03：分类筛选与搜索

**功能描述**: 提供类型 Tab 筛选和全文搜索两种定位方式。Tab 筛选即时切换分组视图，搜索在标题、内容、摘要三个字段中模糊匹配。

**前置条件**:
- 记忆列表已加载

**主要行为**:
1. **Tab 筛选**：点击「全部/画像/规则/纠正」Tab，列表即时切换为对应类型的记忆
2. **搜索**：在搜索框输入关键词，实时过滤标题、内容、摘要中包含关键词的记忆
3. **组合使用**：Tab 筛选和搜索可叠加——先按类型筛选，再在结果中搜索

**业务规则**:
- 默认选中「全部」Tab
- 搜索为前端实时过滤（输入即过滤，无需按回车），匹配字段：title + body + preview
- 搜索不区分大小写
- 搜索与 Tab 筛选叠加时，两者条件取交集
- 搜索框清空时恢复当前 Tab 的完整列表
- 某类型下搜索无结果时，该分组不展示（不展示空分组）

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 搜索无任何匹配 | 所有分组不展示，列表区域展示空状态"未找到匹配的记忆" |
| 搜索框仅输入空格 | 视为空搜索，展示当前 Tab 全部记忆 |
| 快速连续输入 | 每次输入均触发重新过滤（前端计算量小，无需防抖） |

#### F-04：去重聚合

**功能描述**: 新增记忆时，若同类型下已存在相同标题的记忆，触发聚合弹窗。展示已有内容和新增内容，由用户选择处理方式。

**前置条件**:
- 用户正在添加新记忆（非编辑模式）
- 同类型下已存在相同标题的记忆

**主要行为**:
1. 用户提交新增后，系统检测到同类型同标题
2. 弹出「发现同名记忆」弹窗，左侧展示已有内容，右侧展示新增内容
3. 用户选择操作：
   - **追加合并**：将新内容拼接到已有记忆 body 后，evidence 追加发现记录，置信度提升一档
   - **替换内容**：覆盖已有记忆 body，evidence 追加纠正记录，置信度不变
   - **取消**：关闭弹窗，不执行任何操作
   - **仍要新建**：强制创建独立记忆条目（二次确认操作，视觉上用红色虚线按钮暗示谨慎）

**业务规则**:
- 聚合判断条件：`type === existing.type && title === existing.title`
- 不同类型同标题不触发聚合（如"画像:审批习惯"与"规则:审批习惯"可共存）
- 追加合并时 body 以中文分号「；」拼接，preview 取拼接后前 30 字符
- 追加合并自动提升置信度：low → mid → high（每追加一次升一档，high 为上限）
- 替换内容不改变置信度（纠正不代表更确信）
- 聚合操作中若内容含时间模式，同样走定时任务联动逻辑

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 已有记忆为高确，追加合并 | 置信度保持 high，不超出上限 |
| 选择「仍要新建」后再次添加同名 | 再次触发聚合弹窗 |
| 聚合弹窗中点击遮罩层 | 等同于「取消」操作 |

#### F-07：定时任务联动

**功能描述**: 记忆内容中涉及时间相关信息（如"每周二四 10:00-12:00 发布窗口"、"周报周五下班前提交"）时，自动与日程活动模块联动：新增时自动生成定时任务，编辑时同步更新，删除时解除关联。

**前置条件**:
- 记忆内容包含可识别的时间模式
- 日程活动模块可用

**主要行为**:
1. **新增/编辑时检测**：用户在内容输入框输入时，实时检测是否包含时间模式。若检测到，弹窗底部出现紫色提示条：「检测到时间相关内容，保存后将自动生成定时任务并关联到日程活动模块。」
2. **保存时联动**：
   - 新增记忆且含时间模式 → 自动生成一条定时任务（标题取记忆标题，排期待解析），Toast 提示含任务生成信息
   - 编辑已有任务的记忆且仍含时间模式 → 同步更新关联任务标题，Toast 提示「定时任务已同步生效」
   - 编辑已有任务的记忆但不再含时间模式 → 保留任务不自动删除，Toast 提示「原关联定时任务仍保留，如需解除请至日程活动」
3. **列表展示**：含关联任务的记忆行 meta 区域显示紫色 badge（📅 + 排期摘要）
4. **展开详情**：展开后显示关联任务卡片（任务标题 + 排期 + 跳转日程活动链接）
5. **删除联动**：删除含关联任务的记忆时，确认弹窗额外提示任务解关联信息，删除后 Toast 提示包含任务解关联信息

**业务规则**:
- 时间模式检测规则（正则匹配）：
  - `每[周月年][一二三四五六七日天末]?…[\d:：]+`（如"每周二四 10:00-12:00"）
  - `[周月]报`（如"周报"、"月报"）
  - `\d{1,2}[::：]\d{2}`（如"9:30"）
  - 时间段词：上午/下午/凌晨/傍晚/下班前/上班前
  - 周期词：工作日/周末/节假日
- 每条记忆最多关联一个定时任务
- 定时任务关联关系通过记忆的 `linkedTask` 字段维护
- 时间模式检测为提示性质，不阻断保存流程
- 任务自动生成后，Agent 后续提取记忆时若时间信息有误，需同步更新已关联的定时任务并给用户提示

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 内容包含时间词但非真正排期（如"历史上下午班的区分"） | 可能误检，用户可忽略提示正常保存，后续迭代优化 NLP 提取 |
| 记忆被追加合并后新增了时间模式 | 若原无任务则自动生成，若有任务则同步更新 |
| 删除记忆后用户想恢复关联任务 | 任务已被解除关联，需至日程活动手动重新创建 |
| 日程活动模块不可用 | 仅展示提示，不阻断保存，任务生成异步重试 |

#### F-05：证据链展示

**功能描述**: 展开记忆详情时，展示该记忆的来源依据和演变历史。证据链以时间线形式呈现，每条证据标注日期、类型（发现/确认/纠正）和具体内容。置信度仅在行级 meta 信息条中显示，展开详情中不单独展示置信度依据。

**前置条件**:
- 记忆至少有一条 evidence 记录

**主要行为**:
1. 用户点击记忆行展开详情
2. 详情区域展示记忆正文
3. 若有关联定时任务，展示任务卡片
4. 若有 evidence，展示 EVIDENCE 标题 + 时间线列表
5. 每条 evidence 以不同颜色圆点标识类型：发现（蓝）、确认（绿）、纠正（橙）

**业务规则**:
- evidence 类型语义：
  - **发现（discover）**：Agent 从用户行为中观察推断
  - **确认（confirm）**：用户主动认可或 Agent 交叉验证
  - **纠正（correct）**：用户明确纠正 Agent 的错误认知
- evidence 由系统自动记录，用户不可手动编辑
- 追加合并时自动追加 discover 类型 evidence
- 替换内容时自动追加 correct 类型 evidence

#### F-06：删除二次确认

**功能描述**: 删除记忆前弹出确认弹窗，防止误操作。若记忆含关联定时任务或证据链，弹窗需额外提示影响范围。

**前置条件**:
- 用户点击记忆行的删除按钮

**主要行为**:
1. 弹出「确认删除」弹窗
2. 弹窗显示记忆标题
3. 若含关联定时任务，额外提示"关联的定时任务也将一并解除关联"
4. 若无关联任务，提示"此操作不可撤销，关联的证据链也将一并移除"
5. 用户点击「删除」执行删除，点击「取消」关闭弹窗
6. 点击弹窗遮罩层等同于取消

**业务规则**:
- 删除为硬删除，不可恢复
- 删除后自动折叠展开的详情区域
- 删除 Toast 提示包含关联任务解关联信息（如有）

#### F-08：纠正标记

**功能描述**: 当用户发现 Agent 记忆有误时，可通过编辑修正或直接纠正。纠正操作会生成「纠正」类型的记忆条目，并标记来源为纠正，同时在原记忆的 evidence 中追加纠正记录。

**前置条件**:
- 用户发现某条记忆内容有误
- 该记忆存在于记忆库中

**主要行为**:
1. **方式一：直接编辑修正** — 用户编辑原记忆内容，保存后 evidence 追加一条 correct 类型记录，描述纠正内容
2. **方式二：添加纠正记忆** — Agent 自动提取有误时生成纠正类型记忆，标题通常为"XXX纠正"或"XXX门槛"，body 说明原记忆的错误和正确内容
3. 纠正记忆在列表中以 ✧ 图标和橙色视觉标识，与画像/规则分组独立展示

**业务规则**:
- 纠正类型（correct）记忆由系统自动生成或用户在聚合弹窗中选择「替换内容」时产生
- 用户不可在新增弹窗中主动选择「纠正」类型（下拉仅展示画像/规则）
- 纠正记忆的 source 固定为 correct
- 纠正记忆通常关联一条被纠正的原记忆，在 evidence 中记录纠正原因
- 纠正记忆不影响原记忆的置信度（纠正 ≠ 更确信，只是内容修正）

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 用户想删除纠正记忆 | 正常走删除二次确认流程 |
| 纠正记忆又被新的纠正覆盖 | 正常生成新的纠正记忆，原纠正记忆保留为历史记录 |

## 6. 数据模型

### 6.1 TypeScript 接口定义

```typescript
// 记忆类型枚举
enum MemoryType {
  PREF = 'pref',       // 画像
  KNOW = 'know',       // 规则
  CORRECT = 'correct', // 纠正（系统自动生成）
}

// 置信度枚举
enum Confidence {
  HIGH = 'high', // 高确
  MID = 'mid',   // 中确
  LOW = 'low',   // 低确
}

// 来源枚举
enum Source {
  OBSERVE = 'observe',   // 观察
  DECLARE = 'declare',   // 声明
  CORRECT = 'correct',   // 纠正
}

// 证据类型枚举
enum EvidenceType {
  DISCOVER = 'discover', // 发现
  CONFIRM = 'confirm',   // 确认
  CORRECT = 'correct',   // 纠正
}

// 证据条目
interface Evidence {
  date: string;          // 日期（短格式，如 "6/10"）
  type: EvidenceType;    // 证据类型
  text: string;          // 证据描述
}

// 关联定时任务
interface LinkedTask {
  id: string;            // 任务 ID
  title: string;         // 任务标题
  schedule: string;      // 排期描述
}

// 记忆主实体
interface Memory {
  id: string;            // 唯一标识（UUID v4，mock 阶段可用自增整数）
  type: MemoryType;      // 记忆类型
  title: string;         // 标题（最大 50 字符）
  body: string;          // 内容（最大 2000 字符）
  preview: string;       // 摘要（body 前 30 字符）
  conf: Confidence;      // 置信度
  source: Source;        // 来源
  count: string;         // 观察次数描述（如 "6次"，仅 observe 来源时有值）
  lastUpdated: string;   // 最后更新时间（ISO 8601，用于排序展示；创建时等于 createdAt）
  evidence: Evidence[];  // 证据链
  linkedTask?: LinkedTask; // 关联定时任务（可选）
  createdAt: string;     // 创建时间（ISO 8601，创建后不变）
  userId: string;        // 所属用户 ID
}

// 列表响应
interface MemoryListResponse {
  total: number;
  groups: MemoryTypeGroup[];
}

// 类型分组
interface MemoryTypeGroup {
  type: MemoryType;
  label: string;         // 分组显示名
  icon: string;          // 分组图标
  count: number;         // 该类型记忆数
  items: Memory[];       // 按时间倒序排列的记忆列表
}

// 聚合操作请求
interface MergeRequest {
  existingId: string;    // 已有记忆 ID
  type: MemoryType;      // 记忆类型
  title: string;         // 标题
  body: string;          // 新增内容
  action: 'append' | 'replace' | 'force'; // 聚合方式
}

// 时间模式检测结果
interface TimePatternResult {
  detected: boolean;     // 是否检测到时间模式
  patterns: string[];    // 匹配到的时间模式列表
}
```

### 6.2 字段说明

| 字段名 | 类型 | 必填 | 说明 | 约束 |
|--------|------|------|------|------|
| id | string | 是 | 唯一标识 | UUID v4（mock 阶段可用自增整数） |
| type | enum | 是 | 记忆类型 | pref / know / correct |
| title | string | 是 | 标题 | 最大 50 字符，同类型下唯一 |
| body | string | 是 | 内容 | 最大 2000 字符 |
| preview | string | 是 | 摘要 | body 前 30 字符自动生成 |
| conf | enum | 是 | 置信度 | high / mid / low |
| source | enum | 是 | 来源 | observe / declare / correct |
| count | string | 否 | 观察次数 | 如"6次"，仅 observe 来源时有值，declare/correct 时为空 |
| lastUpdated | string | 是 | 最后更新时间 | ISO 8601，用于排序和展示；创建时等于 createdAt，追加/替换/编辑时更新 |
| evidence | array | 是 | 证据链 | 可为空数组 |
| linkedTask | object | 否 | 关联定时任务 | 最多关联一个 |
| createdAt | string | 是 | 创建时间 | ISO 8601，创建后不变 |
| userId | string | 是 | 所属用户 | 关联用户表 |

### 6.3 ERD（实体关系图）

```
[User] 1 --- N [Memory]
[Memory] 0..1 --- 1 [ScheduledTask]
[Memory] 1 --- N [Evidence]
```

**关系说明**:
| 关系 | 类型 | 说明 |
|------|------|------|
| User → Memory | 一对多 | 一个用户拥有多条记忆 |
| Memory ↔ ScheduledTask | 一对一（可选） | 一条记忆最多关联一个定时任务 |
| Memory → Evidence | 一对多 | 一条记忆拥有多条证据记录 |
