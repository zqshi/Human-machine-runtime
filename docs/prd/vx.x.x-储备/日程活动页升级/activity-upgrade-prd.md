## 1. 功能概述

### 1.1 功能定位

「日程活动」是 HMR 平台登录后主页的核心页面之一，承载用户对数字搭档（Agent）自动调度任务的感知与管控。当前版本仅提供只读的任务列表和实时活动时间线，用户无法修改配置、无法查看历史、无法处理故障。本次升级将其从"信息展示"升级为"任务管理中心"，赋予用户完整的配置权、诊断权和治理权。

**升级范围说明**：本次迭代聚焦于「任务配置区」和「健康度检查区」的交互升级；「活动时间线区」（实时 Agent 执行 trace）在线上生产环境已稳定运行，不在本次升级范围内，保持现有实现不变。

### 1.2 目标用户

| 用户类型 | 特征描述 | 核心诉求 |
|---------|---------|---------|
| 数字搭档使用者 | 已配置 Agent 并拥有定时调度任务的企业员工 | 快速了解任务运行状态，及时处理失败任务，调整调度配置 |
| 团队管理者 | 管理多个 Agent 实例和任务的中层管理者 | 掌握团队任务整体健康度，批量处理异常，优化 Token 消耗 |
| 平台运维 | 负责平台稳定性的 SRE/运维人员 | 快速定位连续失败根因，一键修复，降低人工介入成本 |

### 1.3 核心价值

| 维度 | 描述 |
|------|------|
| 用户价值 | 从"只能看"到"能操作"，减少任务故障排查时间 70%，无需联系运维即可自助修复 |
| 业务价值 | 提升平台自主治理能力，降低运维人力投入，减少因任务故障导致的业务中断 |
| 技术价值 | 统一任务管理交互范式，为后续条件触发、审批流等高级调度能力奠定交互基础 |

### 2.3 详细规格

#### F-01：任务编辑

**功能描述**: 用户可通过弹窗表单编辑任务的完整配置，包括名称、描述、调度周期、告警策略和自动重试开关。保存后任务列表实时更新。

**前置条件**:
- 用户已登录且拥有该任务的编辑权限
- 任务存在于当前用户的调度配置中

**主要行为**:
1. 用户点击任务行的编辑按钮（悬停时出现）
2. 弹出编辑弹窗，表单预填充当前任务配置
3. 用户修改任意字段，调度周期区域实时联动预览
4. 点击"保存"，提交修改，弹窗关闭，列表刷新

**业务规则**:
- 任务名称不能为空，最大长度 50 字符
- 任务描述最大长度 200 字符
- 调度周期变更后立即生效（下次执行按新周期调度）
- 编辑弹窗中调度周期切换类型时，自动重置对应配置项

**边界条件**:

| 边界场景 | 处理方式 |
|---------|---------|
| 名称输入为空 | 保存按钮禁用，输入框标红提示"任务名称不能为空" |
| 名称超过 50 字符 | 输入框标红提示"名称不能超过 50 个字符" |
| 编辑过程中任务被其他端删除 | 保存时提示"任务已不存在"，弹窗关闭，列表刷新 |
| 网络异常导致保存失败 | Toast 提示"保存失败，请重试"，弹窗保持打开 |

---

#### F-02：执行历史

**功能描述**: 查看任务的历史执行记录，包含执行时间、状态、耗时、错误信息、重试关系。支持按状态筛选和分页浏览。

**前置条件**:
- 任务存在且有执行记录

**主要行为**:
1. 用户点击任务行的历史按钮
2. 弹出执行历史弹窗，顶部显示统计概览（总次数/成功/失败）
3. 下方显示筛选标签页（全部/成功/失败）和执行记录列表
4. 失败记录点击可展开查看错误详情
5. 记录超过 10 条时显示分页控件

**业务规则**:
- 默认显示全部记录，按时间倒序排列
- 每页最多 10 条记录
- 重试记录标注"重试#N"并关联原始失败记录
- 统计概览始终基于全部记录，不受筛选影响

**边界条件**:

| 边界场景 | 处理方式 |
|---------|---------|
| 任务无执行记录 | 显示"暂无执行记录"空状态 |
| 仅有一条失败记录 | 统计概览正常显示，筛选标签页计数正确 |
| 执行记录超过 100 条 | 分页正常工作，不限制总记录数 |

---

#### F-03：故障内联提示与重试

**功能描述**: 任务上次执行失败时，在任务行内直接显示错误原因摘要和重试按钮，用户无需打开弹窗即可快速处理故障。

**前置条件**:
- 任务 `lastStatus` 为 `failed`
- 任务存在 `lastError` 信息

**主要行为**:
1. 失败任务行下方显示错误信息条，包含错误图标、错误消息文本、重试按钮
2. 用户点击"重试"按钮，触发任务重新执行
3. 重试成功后，错误信息条消失，任务状态更新为"今日已执行"

**业务规则**:
- 错误消息截取 `lastError.message`，最大显示 120 字符，超出省略
- 重试操作不修改任务的调度配置，仅触发一次额外执行
- 连续失败 ≥3 次的任务在健康度检查中标记为"需关注"

**边界条件**:

| 边界场景 | 处理方式 |
|---------|---------|
| 重试请求失败 | Toast 提示"重试失败，请稍后再试" |
| 重试期间用户离开页面 | 后端重试不受影响，下次进入页面状态已更新 |
| 错误信息为空 | 错误信息条显示"执行失败，原因未知" |

---

#### F-04：任务删除

**功能描述**: 删除不再需要的任务，删除前需二次确认，操作不可撤销。

**前置条件**:
- 用户已登录且拥有该任务的删除权限

**主要行为**:
1. 用户点击任务行的删除按钮（悬停时出现）
2. 弹出确认弹窗，显示任务名称和不可撤销警告
3. 用户点击"删除"，任务从列表移除，弹窗关闭

**业务规则**:
- 删除操作不可撤销，确认弹窗必须明确提示
- 删除后任务从调度配置中移除，不再执行
- 删除后列表和统计信息实时更新

**边界条件**:

| 边界场景 | 处理方式 |
|---------|---------|
| 删除正在执行中的任务 | 允许删除，后端负责终止执行 |
| 删除过程中网络异常 | Toast 提示"删除失败，请重试" |

---

#### F-06：健康度详情与批量修复

**功能描述**: 点击健康检查卡片，查看该检查项的详细说明和受影响任务列表，并提供一键批量修复操作。

**前置条件**:
- 存在非"ok"级别的健康检查项

**主要行为**:
1. 用户点击健康检查卡片
2. 弹出详情弹窗，显示检查标题、描述、受影响任务列表
3. 每个受影响任务旁显示针对性操作按钮（如"刷新凭证"、"增大间隔"、"清理"）
4. 弹窗底部提供一键批量操作按钮（如"一键重试失败任务"、"一键清理"）

**业务规则**:
- 不同检查类型提供不同的批量操作：
  - 连续失败 → "一键重试失败任务"
  - 已禁用任务 → "一键清理"
  - 高频轮询 → "批量调整间隔"
- 批量操作执行后，弹窗关闭，Toast 提示操作结果

**边界条件**:

| 边界场景 | 处理方式 |
|---------|---------|
| 无受影响任务 | 不显示受影响任务列表区域 |
| 批量操作部分失败 | Toast 提示"部分任务操作失败，请逐个检查" |

## 4. 交互流程

### 4.1 主流程：编辑任务

```
[起始点] 用户悬停任务行
    ↓
[步骤1] 操作栏淡入显示（历史/执行/编辑/删除）
    ↓
[步骤2] 用户点击"编辑"按钮
    ↓
[步骤3] 编辑弹窗打开，表单预填充当前配置
    ↓
[步骤4] 用户修改字段（调度周期实时预览联动）
    ↓
[判断] 表单校验是否通过？
    ↓ [是]                    ↓ [否]
[步骤5a] 提交保存           [步骤5b] 标红提示校验错误
    ↓                         ↓
[步骤6a] 弹窗关闭           [步骤6b] 用户修正后重新提交
    ↓
[步骤7a] 列表刷新，Toast 提示"已更新「任务名」"
```

**流程说明**:
1. 操作栏仅在悬停时显示，避免视觉噪音
2. 调度周期切换类型时，自动显示对应配置组件（每天→时间选择器，每周→星期芯片+时间，固定间隔→数值+单位）
3. 保存后列表和统计底栏同步刷新

### 4.2 主流程：处理失败任务

```
[起始点] 页面加载，存在 lastStatus=failed 的任务
    ↓
[步骤1] 任务行内显示错误信息条（错误图标 + 消息 + 重试按钮）
    ↓
[判断] 用户选择如何处理？
    ↓ [点击重试]              ↓ [点击编辑]
[步骤2a] 触发重试请求       [步骤2b] 打开编辑弹窗修改配置
    ↓                         ↓
[步骤3a] 重试成功           [步骤3b] 修改后保存
    ↓                         ↓
[步骤4a] 错误条消失，状态更新为"今日已执行"  [步骤4b] 同左
```

### 4.3 主流程：查看执行历史

```
[起始点] 用户点击任务行的"历史"按钮
    ↓
[步骤1] 执行历史弹窗打开
    ↓
[步骤2] 顶部显示统计概览（总次数/成功/失败）
    ↓
[步骤3] 下方显示筛选标签页 + 记录列表
    ↓
[判断] 用户是否点击失败记录？
    ↓ [是]                    ↓ [否]
[步骤4a] 展开错误详情       [步骤4b] 仅浏览
    ↓
[步骤5a] 查看完整错误信息和重试关系
```

### 4.4 异常分支

| 异常场景 | 触发条件 | 处理方式 | 用户提示 |
|---------|---------|---------|---------|
| 网络超时 | 请求超时 >10s | 展示重试按钮或自动重试 | "网络异常，请重试" |
| 权限不足 | 用户无操作权限 | 禁用操作按钮 | 操作按钮灰显，Tooltip 提示"暂无权限" |
| 任务不存在 | 编辑/删除时任务已被移除 | 弹窗关闭，列表刷新 | "任务已不存在" |
| 保存校验失败 | 名称空/超长 | 行内错误提示 | "任务名称不能为空" / "名称不能超过 50 个字符" |
| 批量操作部分失败 | 一键修复中部分任务失败 | Toast 提示部分失败 | "部分任务操作失败，请逐个检查" |
| 并发编辑冲突 | 同一任务被两端同时编辑 | 后提交者覆盖先提交者 | "配置已更新"（乐观更新策略） |

### 4.5 状态机

| 对象 | 状态列表 | 状态转换触发条件 |
|------|---------|---------------|
| 任务执行状态 | 已暂停 → 待执行 → 执行中 → 今日已执行 / 上次失败 | 启用/调度触发/执行完成/执行失败 |
| 任务启用状态 | 启用 ↔ 禁用 | Toggle 开关 / 编辑弹窗 |
| 健康检查级别 | ok → info → warn | 检查条件变化 |

**状态转换图**:
```
[已暂停] --启用--> [待执行]
[待执行] --调度触发--> [执行中]
[执行中] --成功--> [今日已执行]
[执行中] --失败--> [上次失败]
[上次失败] --重试成功--> [今日已执行]
[上次失败] --重试失败--> [上次失败]
[今日已执行] --禁用--> [已暂停]
```

### 附录A：术语表

| 术语 | 定义 |
|-----|------|
| CRON 任务 | 基于预设时间（每天/每周/固定间隔）定时执行的调度任务 |

| 告警策略 | 任务失败时的通知方式：通知（默认推送告警）或静默（仅记录不推送） |
| 自动重试 | 任务失败后自动重新执行，最多重试 3 次 |
| 健康度评分 | 基于任务运行状态综合计算的 0-100 分值，反映调度系统整体健康程度 |
