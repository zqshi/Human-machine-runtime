## 1. 功能概述

### 1.1 功能定位

AI Gateway 是 **HMR 企业级 AI 数字员工平台**「组织控制面」下的**模型接入与治理模块**。HMR 让 AI 从工具变成团队里的数字同事（用 · 造 · 共享），平台上每一个数字同事、Agent、技能的运转都依赖大模型调用——AI Gateway 为这些调用提供统一治理底座：向上为全平台数字员工提供稳定、可授权、可计费、可风控的模型调用通道；向下对接多家大模型供应商（Anthropic、OpenAI、Google、DeepSeek、智谱、通义千问、Mistral 等）。

它集中回答组织控制面对模型能力的五个关切：

- **谁来供、供哪些模型**——模型管理 + 多供应商统一接入
- **谁能用**——模型授权，数字员工白名单治理（对应组织控制面"权限管控"）
- **服务挂了怎么办**——故障转移，保障数字员工不宕机
- **钱花在哪、谁花的**——成本分析，按人/部门/模型精确分账（对应组织控制面"成本管控"）
- **能不能说、该不该拦**——风险规则，内容安全前置拦截（对应组织控制面"安全合规"）

本模块对应 HMR 功能成熟度表中"模型网关与计费（多模型代理 + 精确计费）"。

页面由三大 Tab 组成：**模型管理、成本分析、风险规则**。其中"模型授权"与"故障转移"作为系统级能力内嵌于"模型管理" Tab（前者从模型卡片与编辑抽屉触发，后者置于 Tab 底部）。页面顶部提供全局统计指标（总调用、总 Token、平均延迟、错误率）与统一的日期筛选器。

### 1.2 目标用户

| 用户类型 | 特征描述 | 核心诉求 |
|---------|---------|---------|
| 平台管理员 / 运维 | 负责 AI 基础设施配置与稳定性 | 快速增删改模型、配置授权与故障转移、保障可用性 |
| 成本管理者（财务 / 部门负责人） | 关注 AI 投入产出 | 按部门 / 用户 / 模型看清成本分布与趋势 |
| 安全合规人员 | 关注内容合规与数据外泄 | 配置并测试风险拦截规则，防止敏感信息泄露 |
| 数字同事使用者（间接受益） | 通过协作工具 / 配置中心使用数字同事 | 感知不到底层模型切换与授权，享受稳定、安全的 AI 服务 |

### 1.3 核心价值

| 维度 | 描述 |
|------|------|
| 用户价值 | 管理员在单一面板完成 HMR 全平台数字员工所依赖的模型管理、授权治理、成本洞察与安全风控，免去在多个供应商控制台之间切换 |
| 业务价值 | 支撑 HMR 核心主张「敢用」——白名单授权让模型能力按需开放、风险可控；故障转移保障数字同事服务连续性；成本按人/部门精确分账，让组织敢于大规模铺开数字员工；风险规则前置拦截，降低 AI 接触核心业务时的数据外泄与合规风险 |
| 平台价值 | 作为组织控制面的模型治理底座，统一接入、授权与计费抽象，支撑 HMR 控制面服务体系；新供应商接入仅需配置，无需改造数字员工的调用链路 |

#### F-01：模型管理

**功能描述**: 以卡片网格形式展示全部模型，支持模型的创建、编辑、删除、启用/停用与健康检查。平台已登记的模型与外部代理已纳管的模型统一并入列表展示，外部代理纳管的模型默认启用。

**前置条件**:
- 用户已登录管理后台并具备管理员权限
- 已进入「AI Gateway → 模型管理」Tab

**主要行为**:
1. 进入 Tab 自动加载模型列表，顶部显示"N 个模型"计数
2. 点击「添加模型」打开右侧编辑抽屉，填写配置后保存即新增
3. 点击卡片「编辑」打开抽屉回填现有配置，修改后保存
4. 点击卡片「删除」弹出确认框，确认后删除
5. 点击卡片右上角开关切换模型启用/停用状态
6. 点击卡片「健康检查」按钮触发检测，刷新该模型的健康状态与检测时间
7. 点击卡片「已授权 N 个 Agent」或编辑抽屉内「授权数字员工」进入授权面板（见 F-06）

**模型卡片信息呈现**:
- 健康状态圆点（颜色指示 + 提示：状态·延迟·检测时间）+ 显示名称 + 健康检查中"检测中…"提示 / 延迟数值
- 描述文字
- 标签行：供应商类型标签 / 供应商模型名（等宽字体）/ 模型别名（蓝色标签）
- 价格行：输入价格（$/M）/ 输出价格（$/M）/ 上限（Token）/ 限流（/min）
- 底部左侧：「已授权 N 个 Agent」徽标按钮（点击进入授权面板）
- 底部右侧操作区：健康检查 / 编辑 / 删除

**业务规则**:
- 模型完整配置分六组：基础配置、连接配置、性能配置、价格配置、启用开关、授权数字员工
- 选择供应商时自动联动填充 Base URL 与协议类型（仅新建时）
- 新建模型协议类型默认为 Anthropic
- 模型计价单位统一为"每百万 Token（/M tok）"
- 货币仅支持 USD 与 CNY
- 限流单位为"请求/分钟（req/min）"，默认 60
- 输入/输出价格默认 0
- 编辑模型时 API Key 留空表示不更新原密钥
- 外部代理纳管的模型：未在本平台登记的自动并入列表，默认启用

**配置项清单（产品字段）**:

| 配置组 | 字段 | 必填 | 说明 / 约束 |
|--------|------|------|------------|
| 基础配置 | 供应商 | 是 | 下拉选择，含"自定义"选项 |
| 基础配置 | 显示名称 | 是 | 用户可读名称 |
| 基础配置 | 模型别名 | 否 | 客户端请求时使用的模型名 |
| 基础配置 | 描述 | 否 | 模型用途说明 |
| 基础配置 | 协议类型 | 是 | OpenAI Compatible / Anthropic / Google（新建默认 Anthropic） |
| 连接配置 | Base URL | 是 | API 基础地址 |
| 连接配置 | 供应商模型名 | 是 | 供应商侧模型标识 |
| 连接配置 | API Key | 新建必填 | 密钥；编辑可留空（不更新） |
| 性能配置 | 最大 Token | 否 | 若填须为正整数 |
| 性能配置 | 限流 (req/min) | 否 | 默认 60，若填须为正整数 |
| 性能配置 | 请求超时 (秒) | 否 | 若填须为正整数 |
| 性能配置 | 流式超时 (秒) | 否 | 若填须为正整数 |
| 价格配置 | 输入价格 (/M tok) | 否 | 不能为负，默认 0 |
| 价格配置 | 输出价格 (/M tok) | 否 | 不能为负，默认 0 |
| 价格配置 | 货币 | 是 | USD / CNY |
| 价格配置 | 缓存读取价格 (/M tok) | 否 | — |
| 价格配置 | 缓存创建价格 (/M tok) | 否 | — |
| 开关 | 启用 | 是 | 默认启用 |
| 授权 | 授权数字员工 | 否 | 保存模型后方可配置白名单（见 F-06） |

**边界条件**:

| 边界场景 | 处理方式 |
|---------|---------|
| 模型列表为空 | 展示"暂无模型"空状态 |
| 必填项为空 | 提交时校验拦截，对应字段红框 + 行内提示 |
| 数值字段非正整数 | 提示"必须为正整数" |
| 价格为负数 | 提示"不能为负数" |
| 健康检查进行中 | 按钮禁用并显示加载图标，卡片显示"检测中…" |
| 删除操作 | 二次确认弹窗"确定要删除该模型配置吗？" |

#### F-02：故障转移

**功能描述**: 作为系统级高可用能力，配置一条全局生效的故障转移规则。当主模型发生异常（欠费 / 限流 / 服务不可用 / 超时 / 报错）时，按预设策略自动切换至备用模型，对终端用户无感知。

**前置条件**:
- 已存在至少一个可选模型（模型管理中已配置模型）

**主要行为（编辑态）**:
1. 点击「配置规则 / 编辑」进入编辑态
2. 选择主模型（下拉，必填）
3. 选择切换策略：指定顺序（ordered）/ 随机切换（random）
4. 从候选区勾选模型加入备用链，已选区可移除
5. 指定顺序策略下，可拖拽已选项调整优先级
6. 保存即生效（规则自动启用）

**主要行为（展示态）**:
- 展示主模型名称、切换策略、备用模型链
- 指定顺序策略：以"1.A → 2.B → 3.C"形式展示优先级
- 随机切换策略：以"A、B、C"形式展示

**业务规则**:
- 规则全局唯一，保存后即启用
- 主模型为必填项，未选择时提示"请选择主模型"
- 备用模型与主模型互斥（主模型不会出现在备用候选中）
- 已选备用模型也不会重复出现在候选区
- 备用模型为空时，规则保存但不生效，展示态提示"未配置，故障转移不会生效"
- 未配置任何规则时，提示"尚未配置故障转移。主模型异常时调用将直接失败。"

**边界条件**:

| 边界场景 | 处理方式 |
|---------|---------|
| 无任何模型可选 | 候选区提示"无可添加的备用模型"，主模型下拉为空 |
| 已选备用为空 | 提示"尚未选择备用模型" |
| 主模型未选 | 校验拦截，提示"请选择主模型" |

#### F-03：成本分析〔本期暂不支持·延后下期〕

**功能描述**: 从部门、用户、模型、日期四个维度统计 AI 调用成本，配合全局指标卡，帮助管理者看清"钱花在哪、谁花的、哪个模型花的、趋势如何"。时间范围复用页面级日期筛选器。

**前置条件**:
- 已进入「成本分析」Tab
- 已有 AI 调用产生的成本数据

**主要行为**:
1. 进入 Tab 自动按当前日期范围加载统计数据
2. 顶部 4 个指标卡展示全局总量：输入 Token / 输出 Token / 总成本 / 缓存命中
3. 下方依次展示四张汇总表：部门汇总、用户汇总、模型汇总（与日趋势并排）
4. 用户汇总支持按部门筛选、按成本/Token/调用排序
5. 切换页面日期范围后，所有统计自动刷新

**统计维度明细**:

| 汇总表 | 展示列 | 备注 |
|--------|--------|------|
| 全局指标卡 | 输入 Token / 输出 Token / 总成本(¥) / 缓存命中 | 顶部一行四卡 |
| 部门汇总 | 部门 / 人数 / 调用 / Tokens / 成本(¥) / 占比 | 占比带进度条可视化 |
| 用户汇总 | 用户 / 部门 / 调用 / Tokens / 成本(¥) | 支持部门筛选 + 三字段排序 |
| 模型汇总 | 模型 / 调用 / Tokens / 缓存Token / 成本(¥) | 与日趋势并排 |
| 日趋势 | 日期 / 调用 / 入/出Token / 缓存Token / 成本(¥) | 与模型汇总并排 |

**业务规则**:
- 成本金额统一以人民币（¥）展示
- Token 数量自动按 K / M 简写（≥1000 显示 K，≥100 万显示 M）
- 部门汇总"占比"= 该部门成本 / 全部成本 × 100%
- 缓存 Token 为 0 时以"—"显示，不显示 0
- 各表无数据时统一展示"暂无数据"

**边界条件**:

| 边界场景 | 处理方式 |
|---------|---------|
| 所选时间范围内无调用 | 各表展示"暂无数据"，指标卡显示 0 或 — |
| 某部门无用户明细 | 用户汇总该部门行为空 |
| 缓存 Token 为 0 | 显示"—" |

#### F-04：风险规则〔本期暂不支持·延后下期〕

**功能描述**: 提供内容安全风控规则库管理，通过正则表达式匹配识别风险内容（如安全凭证、个人隐私、公司信息等），命中后按预设动作处置（拦截 / 安全路由 / 放行）。支持分类筛选、规则测试与导出。

**前置条件**:
- 已进入「风险规则」Tab
- 具备规则管理权限

**主要行为**:
1. 进入 Tab 自动加载规则列表，按分类按钮组筛选（含计数）
2. 点击「新建规则」打开编辑抽屉填写配置
3. 点击规则卡片「编辑」回填配置修改
4. 点击规则卡片开关切换启用/停用
5. 点击「删除」二次确认后删除
6. 在底部"规则测试"区输入文本（支持多行），点击「测试」返回命中结果
7. 点击「导出」下载全部规则为 JSON 文件

**规则配置项（产品字段）**:

| 字段 | 必填 | 说明 / 约束 |
|------|------|------------|
| 规则 ID | 是 | snake_case 格式，创建后不可修改（只读） |
| 名称 | 是 | 规则可读名称 |
| 描述 | 否 | 规则用途说明 |
| 分类 | 是 | 安全凭证(security) / 个人隐私(privacy) / 公司信息(company) / 自定义(custom) |
| 正则表达式 | 是 | 风险匹配模式 |
| 严重度 | 是 | 低(low) / 中(medium) / 高(high)，默认中 |
| 命中动作 | 是 | 放行(allow) / 安全路由(route_secure_model) / 拦截(block)，默认安全路由 |
| 启用开关 | 是 | 控制规则是否生效 |

**规则卡片信息呈现**:
- 名称 + 启用开关
- 描述
- 正则表达式（等宽字体，灰底，截断显示）
- 标签组：分类 / 严重度（低灰/中黄/高红）/ 命中动作（放行绿/安全路由蓝/拦截红）
- 操作区：编辑 / 删除

**业务规则**:
- 规则 ID 命名须为 snake_case，且创建后不可修改
- 规则 ID 与正则表达式为必填项，缺一不可保存
- 命中动作为三层处置：放行（记录但通过）/ 安全路由（转安全模型处理）/ 拦截（直接阻止）
- 分类筛选按钮组每项显示该分类规则计数，含"全部"项
- 规则测试支持多行文本批量测试，仅对启用中规则做正则匹配，返回命中条数；无命中时提示"未检测到风险，所有规则均通过"

**边界条件**:

| 边界场景 | 处理方式 |
|---------|---------|
| 规则列表为空 | 展示"暂无规则"空状态 |
| 某分类下无规则 | 该分类筛选结果为"暂无规则" |
| 测试文本为空 | 不发起测试 |
| 测试有命中 | 红色提示框，显示"检测到 N 条风险命中" |
| 测试无命中 | 绿色提示框，显示"未检测到风险，所有规则均通过" |
| 删除操作 | 二次确认弹窗"确定要删除该风控规则吗？" |

#### F-06：模型授权（数字员工白名单）

**功能描述**: 为每个模型配置"可调用该模型的数字员工（Agent）白名单"。授权以白名单语义生效：未被勾选的数字员工将无法调用该模型。授权面板采用**声明式三维授权**（按部门级联继承 / 按成员 / 按实例），以组织结构为主视图，交互细节见附录D。模型卡片底部与模型编辑抽屉均可进入授权面板。

**前置条件**:
- 模型已创建（新建态下授权入口禁用，提示"保存模型后可配置授权（白名单 · 默认关闭）"）
- 存在可被授权的数字员工（实例）数据

**主要行为**:
1. 从模型卡片「已授权 N 个 Agent」按钮，或编辑抽屉「授权数字员工」组，打开右侧授权抽屉
2. 顶部提示授权语义（白名单 · 默认关闭：未勾选的数字员工无法调用）
3. 通过搜索框按名称 / 部门 / 负责人筛选数字员工
4. 切换分组维度：按部门 / 按负责人（员工）
5. 整组勾选（分组头部三态：未选 / 部分 / 全选），或逐个勾选
6. 提供"全选筛选结果"与"清空"快捷操作
7. 保存授权，返回卡片刷新授权计数

**授权面板信息呈现**:
- 顶部：白名单语义提示条（amber）+ 搜索框 + 分组维度切换（按部门 / 按员工）
- 列表：按所选维度分组，分组头显示分组名 + 已选/总数，支持三态整组勾选
- 列表项：勾选框 + 实例名称 + 负责人 + 运行状态（运行中/已停止徽标）；含"共享"的实例标注"共享"标签
- 底部：已选 N 个 Agent + 取消 / 保存授权

**业务规则**:
- 授权为白名单语义：保存后仅被勾选的数字员工可调用该模型，其余被拒绝
- 模型卡片「已授权 N 个 Agent」实时反映授权规模

**边界条件**:

| 边界场景 | 处理方式 |
|---------|---------|
| 新建模型（未保存） | 授权入口禁用，提示"保存模型后可配置授权（白名单 · 默认关闭）" |
| 无可授权数字员工 | 列表展示"没有匹配的数字员工" |
| 全清授权后保存 | 该模型对全部数字员工关闭，卡片计数归 0 |

## 4. 交互流程

### 4.1 主流程

**流程一：新增模型**

```
[入口] 管理员进入「模型管理」Tab
    ↓
[操作] 点击「添加模型」
    ↓
[表单] 右侧抽屉打开，逐组填写：基础配置 → 连接配置 → 性能配置 → 价格配置 → 启用开关
    ↓
[校验] 提交时校验必填项与格式
    ↓ 校验通过            ↓ 校验失败
[保存] 新增模型          [提示] 对应字段红框+行内提示，停留表单
    ↓
[刷新] 列表刷新，新卡片出现（授权入口此时可点）
```

**流程二：配置模型授权**

```
[入口] 模型卡片「已授权 N 个 Agent」或编辑抽屉「授权数字员工」
    ↓
[面板] 右侧授权抽屉打开，展示全部数字员工（按部门/员工分组）
    ↓
[筛选/分组] 搜索关键字 / 切换按部门·按员工维度
    ↓
[勾选] 整组勾选 或 逐项勾选（白名单语义）
    ↓
[保存] 保存授权集
    ↓
[刷新] 卡片「已授权 N 个 Agent」计数更新
```

**流程三：配置故障转移**

```
[入口] 模型管理 Tab 底部「故障转移」区块
    ↓
[操作] 点击「配置规则」（首次）/「编辑」
    ↓
[选择] 选择主模型（必填）
    ↓
[选择] 选择切换策略（指定顺序 / 随机切换）
    ↓
[选择] 从候选区勾选备用模型 → 已选区
    ↓ 指定顺序策略        ↓ 随机切换策略
[排序] 拖拽调整优先级     [跳过排序]
    ↓
[保存] 规则生效（自动启用）
    ↓
[展示] 切换为展示态，显示主模型/策略/备用链
```

**流程四：风险规则测试**

```
[入口] 「风险规则」Tab 底部「规则测试」区
    ↓
[输入] 在文本框输入待测文本（支持多行）
    ↓
[操作] 点击「测试」
    ↓
[判断] 是否有启用中规则命中
    ↓ 有命中              ↓ 无命中
[红色提示] 显示"检测到N条命中"  [绿色提示] "未检测到风险，所有规则均通过"
```

**流程说明**:
1. 模型新增/编辑共用同一抽屉，依据是否携带模型数据区分新增/编辑态；授权入口仅对已保存模型可用
2. 模型授权与故障转移均为展示/编辑双态或抽屉式，保存后即时刷新
3. 规则测试结果即时展示在测试框下方，不离开当前页面

### 4.2 异常分支

| 异常场景 | 触发条件 | 处理方式 | 用户提示 |
|---------|---------|---------|---------|
| 网络异常/请求失败 | 接口超时或错误 | 静默捕获保留当前界面（健康检查等关键操作走 toast） | "健康检查失败，请稍后重试" 等 |
| 必填项为空 | 提交表单未填必填字段 | 校验拦截，禁止提交 | "请输入显示名称" / "请选择主模型" |
| 数值格式错误 | 非正整数 / 负数 | 校验拦截 | "必须为正整数" / "不能为负数" |
| 数据为空 | 列表无数据 | 展示空状态 | "暂无模型" / "暂无规则" / "暂无数据" / "没有匹配的数字员工" |
| 删除确认 | 危险操作 | 二次确认弹窗 | "确定要删除该模型配置吗？" / "确定要删除该风控规则吗？" |
| 健康检查中 | 检测进行中 | 按钮禁用 | "检测中…" + 加载图标 |
| 健康检查未配置 | 模型未配置连接地址 | 状态置"未配置" | "健康检查失败：模型未配置 baseUrl" |
| 故障转移备用为空 | 保存但无备用模型 | 规则保存但不生效 | "未配置，故障转移不会生效" |

### 4.3 状态机

| 对象 | 状态列表 | 状态转换触发条件 |
|------|---------|---------------|
| 模型健康状态 | 未知 → 正常 / 降级 / 异常 / 未配置 | 触发健康检查后据检测结果更新（无连接地址→未配置，不可达→异常，可用但降级→降级，可用→正常） |
| 模型启用状态 | 启用 ⇄ 停用 | 卡片开关切换 |
| 模型授权 | 未配置(0) ⇄ 已授权(N) | 授权面板保存 |
| 风险规则状态 | 启用 ⇄ 停用 | 卡片开关切换 |
| 故障转移规则 | 未配置 → 已配置(启用) | 首次保存规则 |
| 故障转移区块 | 展示态 ⇄ 编辑态 | 点击「编辑/配置」进入，保存/取消退出 |
| 测试结果 | 未测试 → 有命中 / 无命中 | 点击「测试」后返回 |

**状态转换图（模型健康状态）**:
```
[未知] --[健康检查]--> [正常] / [降级] / [异常] / [未配置]
  · 未配置连接地址   → [未配置]
  · 服务不可达/超时  → [异常]
  · 服务可用但降级   → [降级]
  · 服务正常         → [正常]
（任一终态均可通过再次健康检查回到其他终态）
```

### 6.1 业务对象清单

| 业务对象 | 产品含义 | 核心作用 |
|---------|---------|---------|

| 供应商（Provider） | AI 模型服务提供商 | 提供预置接入模板（端点、协议），模型归属于其一 |
| 数字员工（Agent / Instance） | 平台上一个运行中的数字同事实例 | 模型授权的对象（白名单主体） |
| 模型授权（Model Grant） | 模型与数字员工的白名单关系 | 控制哪些数字员工可调用某模型 |
| 故障转移规则（Failover Rule） | 全局唯一的主备切换规则 | 保障主模型故障时的服务连续性 |
| 成本记录（Cost Record） | 单次或聚合的 AI 调用成本 | 成本分析与预算核算的数据基础 |
| 风险规则（Risk Rule） | 内容风控匹配规则 | 前置识别并处置风险内容 |

### 6.2 关键业务属性（产品字段）

**模型（Model）**
- 身份：显示名称、模型别名、描述
- 归属：供应商类型、协议类型、Base URL、供应商模型名、API Key
- 性能：最大 Token、限流（req/min）、请求超时、流式超时
- 价格：输入价格、输出价格、货币、缓存读取价格、缓存创建价格（均为 /M tok）
- 状态：启用/停用、健康状态（正常/降级/异常/未配置/未知）、最近健康检查时间、来源（本平台登记 / 外部代理纳管）

**数字员工（Agent / Instance）**
- 身份：名称、所属租户、所属部门、负责人、运行状态（运行中/已停止）、是否组织级共享

**模型授权（Model Grant）**
- 关系：模型 ↔ 数字员工集合（白名单）
- 语义：白名单（默认关闭，未授权不可调用）
- 规模：已授权数字员工数量

**故障转移规则（Failover Rule）**
- 主模型、切换策略（指定顺序/随机切换）、备用模型链、启用状态

**风险规则（Risk Rule）**
- 规则 ID（snake_case，不可改）、名称、描述、分类、正则表达式、严重度、命中动作、启用状态

**成本记录（Cost Record）**
- 维度：模型、用户、部门、日期
- 度量：输入 Token、输出 Token、缓存 Token、调用次数、估算成本

### 6.3 对象关系

```
供应商 1 ──── N 模型
模型   N ──── N 数字员工（通过"模型授权"白名单关联）
模型   N ──── N 调用（产生）成本记录
故障转移规则 1 ──* 包含 1 主模型 + N 备用模型（引用模型）
用户/负责人 1 ──── N 数字员工   部门 1 ──── N 数字员工
用户 1 ──── N 成本记录   部门 1 ──── N 用户
```

**关系说明**:

| 关系 | 类型 | 说明 |
|------|------|------|
| 供应商 → 模型 | 一对多 | 一个供应商下可配置多个模型 |
| 模型 ↔ 数字员工 | 多对多 | 通过模型授权白名单关联，决定数字员工可用模型集合 |
| 模型 → 成本记录 | 一对多 | 每次模型调用产生一条成本记录 |
| 故障转移规则 → 模型 | 引用 | 规则引用 1 个主模型 + N 个备用模型 |
| 部门/负责人 → 数字员工 → 成本记录 | 一对多链 | 成本可按数字员工归属的部门/负责人聚合 |

### 6.4 模型应用与切换规则

> 本节从产品视角说明：数字员工的一次 AI 调用，模型如何被选定、授权如何把关、模型出问题时如何保障不中断。

#### 一次调用，模型从哪来

数字员工发起一次 AI 对话时，系统按以下顺序确定使用的模型：

1. **显式指定优先**：若该数字员工或对话明确指定了模型，使用指定模型
2. **默认模型兜底**：未指定时，使用平台默认模型（例如默认 glm-5.1）
3. **统一接入**：无论哪家供应商（Anthropic、OpenAI、Google、DeepSeek、通义千问等），平台通过统一网关接入，对数字员工屏蔽供应商差异——数字员工只需"用模型"，不必关心模型来自哪个供应商
4. **用量留痕**：每次调用的 Token 消耗、耗时、所用模型均被记录，作为成本分析与运行监控的依据

#### 授权把关：谁能用哪个模型

模型授权采用**白名单**机制：默认情况下数字员工无权使用任何模型，只有被管理员明确授权的数字员工才能调用对应模型。

- **授权维度**（F-06）：管理员可按**部门**（勾选部门即授权其全员，含下属子部门，调岗/入职自动跟随）、**成员**（按负责人，名下数字员工自动继承）、**实例**（单个数字员工）三个维度授权，详见附录D
- **未授权的后果**：未被授权的数字员工尝试调用该模型时，调用被拒绝
- **灰度开关**：为降低上线风险，授权校验支持灰度——可先"只记录不拦截"观察影响面，再切换为"强制拦截"（后期支持，本期暂不考虑）
- **平台级豁免**：平台自身的统一助手等不归属具体数字员工的调用，不受模型授权约束（后期支持，本期暂不考虑）

#### 模型不可用：自动故障转移

当主模型出现异常（欠费、限流、服务不可用、超时、报错）时，系统自动切换到备用模型，对数字员工和终端用户无感知：

- **切换策略**：
  - **指定顺序**：按管理员配置的优先级依次尝试备用模型（1 号 → 2 号 → 3 号），首个可用者接管
  - **随机切换**：在备用模型中随机选一个，避免某个备用模型流量过载
- **熔断保护**：某个模型连续失败时，系统暂时"熔断"它（短期内不再尝试），避免持续冲击已故障的模型，一段时间后再试探性恢复
- **自动重试与兜底**：对偶发错误自动重试；若主模型与全部备用模型都不可用，调用失败并告警（服务端配置一个告警机制，后期平台维护告警规则），由管理员介入
- **无备用不生效**：未配置任何备用模型时，故障转移规则保存但不生效——主模型异常时调用直接失败（界面提示"未配置，故障转移不会生效"）

#### 健康状态：模型可用吗

管理员可在模型卡片手动触发健康检查，系统实时探测模型是否可用，并以颜色直观呈现：

| 状态 | 含义 | 颜色 |
|------|------|------|
| 正常 | 模型可用，响应正常 | 绿色 |
| 降级 | 模型可调用但近期不稳定（处于熔断恢复试探中） | 黄色 |
| 异常 | 模型不可达（超时 / 报错 / 服务不可用） | 红色 |
| 未配置 | 未填写接入地址，无法探测 | 灰色 |

健康状态既用于管理员运维感知，也作为故障转移选择备用模型时的参考——异常或熔断中的模型不参与切换。

### 附录D：模型授权能力（声明式三维授权）

> 本期模型授权（F-06）采用"声明式三维授权"语义：以组织结构（部门树）为主视图，支持按部门、按成员、按实例三个维度授权，实现调岗/入职自动跟随。

**设计目标**：以"按组织结构声明式授权"取代逐个实例勾选，实现调岗/入职自动跟随，降低大规模数字员工场景下的授权维护成本。

**三维授权主体**：
- 部门级：勾选某部门，即授权该部门**及其全部下属子部门**的成员
- 用户级：按负责人授权，其名下所有数字员工自动继承
- 实例级：兜底，对单个数字员工单独授权

**期望交互形态**：
- 以组织结构（部门树）为主视图，顶部"显示成员"开关控制是否钻取到成员层
- 部门三态勾选：未授权 / 部分选中（下属部分已选）/ 全选（本部门及全部下属子树已选）；勾选父部门即勾选整棵子树，取消父部门即清空整棵子树
- 继承标注：被部门或用户授权覆盖的成员（用户及其名下实例）显示"继承自 XX"，且不可单独取消（继承不可绕过，需到对应部门/用户节点取消）
- 组织级共享 Agent 独立分组，不归属任何部门，只能实例级单独授权
