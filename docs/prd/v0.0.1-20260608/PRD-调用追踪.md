## 1. 功能概述

### 1.1 功能定位

调用追踪是管理后台 AI 运维域的核心可观测性页面，负责展示平台所有 LLM 调用（含风险审查、工具执行）的完整生命周期。它解决"一次用户请求到底发生了什么"的问题——从调用入口、模型推理、工具调用到最终响应，每一环的耗时、状态、Token 消耗和成本都清晰可见。当调用失败或被拦截时，管理员能快速定位根因环节，而不是在日志大海捞针。

### 1.2 目标用户

| 用户类型 | 特征描述 | 核心诉求 |
|---------|---------|---------|
| 平台管理员 | 负责数字员工平台日常运维，关注调用量和异常率 | 快速发现异常调用，定位根因，保障服务可用性 |
| AI 运维工程师 | 深入关注模型性能、成本和链路瓶颈 | 分析各环节耗时分布，优化关键路径，控制 Token 成本 |
| 安全合规人员 | 关注敏感信息泄露、风险拦截 | 查看风险命中详情，确认拦截是否生效，审计合规 |

### 1.3 核心价值

| 维度 | 描述 |
|------|------|
| 用户价值 | 从"逐条翻日志"到"一键展开瀑布图"，异常定位时间从分钟级降到秒级 |
| 业务价值 | 降低故障 MTTR 50%，Token 成本可见可控，避免隐性超支 |
| 战略价值 | 统一 Trace/Span 追踪模型，兼容单次调用和分布式链路两种场景，为后续告警和自动诊断奠基 |

### 2.3 详细规格

#### F-01：调用统计概览

**功能描述**: 页面顶部以四个统计卡片呈现全局调用状况。

**前置条件**:
- 用户已登录管理后台
- 数据源可用

**主要行为**:
1. 页面加载时自动获取统计数据
2. 展示四个卡片：总调用（蓝色）、已完成（绿色）、失败/错误（红色）、总 Token（紫色）
3. 数值超过 1,000 显示为 `1K`，超过 1,000,000 显示为 `1.0M`
4. 筛选条件变更或刷新时，统计卡片同步更新

**业务规则**:
- 已完成 = 状态为"成功"或"已完成"的记录数
- 失败/错误 = 状态为"错误"或"失败"的记录数
- 总 Token = 所有记录的输入 Token + 输出 Token 之和
- 统计范围不随列表分页变化，始终是全量汇总

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 数据源不可用 | 四个卡片均展示 `0`，不弹错误提示 |
| 数值为 0 | 展示 `0`，不隐藏卡片 |

#### F-02：调用列表与筛选

**功能描述**: 以表格形式展示调用记录列表，支持多维度筛选和分页浏览。

**前置条件**:
- 用户已登录管理后台

**主要行为**:
1. 默认按创建时间倒序展示最近 20 条调用记录
2. 表格列：展开箭头 | 追踪标识 | 调用者 | 状态 | 根因/链路 | 模型 | Tokens | 成本 | 延迟 | 时间
3. 关键词搜索支持匹配追踪标识、请求标识、会话标识、用户、数字员工、模型名称、扩展信息等字段
4. 状态下拉筛选支持：全部 / 成功 / 错误 / 失败 / 已拦截
5. 日期范围筛选：开始日期 + 结束日期
6. 清空按钮：一键重置所有筛选条件
7. 刷新按钮：手动触发数据重新加载
8. 分页：上一页 / 下一页 + 当前页/总页数 + 总记录数

**业务规则**:
- 每页固定 20 条
- 筛选条件变更自动重置到第 1 页
- 追踪标识截断显示前 16 位，蓝色等宽字体
- 调用者优先显示数字员工名称，其次用户标识，均无则显示灰色斜体 "default"
- 状态用彩色圆角标签（Pill）展示：成功=绿、错误/失败=红、已拦截=橙、其他=灰
- 成本格式：`¥0.0450`，无法计算时显示 `—`
- 延迟格式：< 1s 显示 `xxms`，≥ 1s 显示 `x.xs`
- 时间格式：`MM/DD HH:mm:ss`

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 列表为空 | 表格居中显示"暂无调用记录" |
| 筛选无结果 | 同空状态处理 |
| 搜索关键词为空 | 等同不搜索，展示全量 |

#### F-03：链路瀑布图

**功能描述**: 展开某条调用记录后，以瀑布图形式展示该调用下所有链路节点的时序关系、耗时和层级。

**前置条件**:
- 用户点击了列表中某一行
- 该调用存在链路节点数据

**主要行为**:
1. 点击列表行，行下方展开详情区域
2. 再次点击同一行，收起详情区域
3. 展开时自动加载该调用的详情数据（含链路节点列表、风险命中等）
4. 瀑布图顶部展示时间轴标尺，根据总耗时自动选择合适的刻度
5. 每个链路节点占一行：
   - 左侧展示操作名称（按层级缩进，子节点前加树形连接符）
   - 中间展示时间条（颜色对应节点状态，长度对应耗时占比）
   - 右侧展示耗时数值
6. 点击某个节点行，该行高亮，同时下方展开节点详情面板
7. 如果该调用属于更大范围的分布式链路，自动关联并合并展示完整链路

**业务规则**:
- 节点状态颜色：成功/完成=绿色、失败/错误=红色、已拦截=琥珀色、跳过=灰色
- 时间条最小宽度确保极短耗时也可见
- 时间轴刻度根据总耗时自适应
- 加载中显示"正在加载详情..."
- 加载失败显示错误原因，提示"列表摘要仍可用，但无法展示完整链路"

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 链路节点数据为空 | 降级展示简易时间线（F-08） |
| 分布式链路关联失败 | 仅展示当前调用详情，不阻塞页面 |
| 同一条记录重复点击展开 | 收起再展开，不重复加载 |

#### F-04：链路节点详情面板

**功能描述**: 瀑布图中选中某个节点后，展示该节点的完整信息。

**前置条件**:
- 瀑布图已渲染
- 用户点击了某个节点行

**主要行为**:
1. 展示节点基本信息：节点标识、操作名称、耗时、开始时间、状态
2. 如果节点关联了模型，展示模型名称
3. 如果节点有标签（Tags），以键值对网格展示
4. 如果节点有输入内容（Input），默认折叠，点击"▸ 展开 Input"查看
5. 如果节点有输出内容（Output），默认折叠，点击"▸ 展开 Output"查看
6. 内容以格式化方式展示，区域高度有限制，超出可滚动

**业务规则**:
- 节点标识以等宽字体展示
- JSON 格式的内容自动格式化缩进
- 输入/输出默认折叠，避免页面过长

---

#### F-05：风险命中展示

**功能描述**: 展示该调用被风险规则命中的详情列表。

**前置条件**:
- 调用详情已加载
- 该调用存在风险命中记录

**主要行为**:
1. 如果有风险命中，在详情区底部展示红色背景的风险命中区块
2. 标题显示"风险命中 (N)"
3. 每条命中展示：规则名称 — 匹配摘要 [严重级别/处置动作]

**业务规则**:
- 无风险命中时不显示该区块
- 严重级别和处置动作用方括号括起，如 `[high/block]`
- 列表按命中时间正序排列

---

#### F-06：成本与模型信息

**功能描述**: 展示该调用的模型选择和 Token 消耗/成本明细。

**前置条件**:
- 调用详情已加载

**主要行为**:
1. 展示请求模型和实际使用的模型
2. 展示输入 Token 数和输出 Token 数
3. 展示输入成本和输出成本（人民币）

**业务规则**:
- 成本格式：`¥0.0450`（四位小数），无数据时显示 `—`
- 请求模型与实际模型不同时，两者均展示（可能存在模型降级/故障转移）
- 模型名称直接展示原始值，不做映射

---

#### F-07：诊断结论卡片

**功能描述**: 基于调用链路数据自动生成诊断分析，帮助管理员快速理解异常原因。

**前置条件**:
- 调用详情已加载

**主要行为**:
1. 展示三列诊断卡片：
   - **诊断结论**：根因定位（首个失败/错误/被拦截的链路节点），无失败时显示"未识别失败节点"；影响范围（关联的任务、数字员工或用户）
   - **关键路径**：按耗时降序排列的 Top 链路节点（最多 3 条）
   - **风险与建议**：风险命中数量 + 针对性建议
2. 如果链路数据存在质量问题（缺少节点、缺少任务标识、缺少调用者信息），展示黄色警告提示

**业务规则**:
- 根因判定：从链路节点中寻找首个失败/错误/被拦截的节点；若无明确失败节点但整体状态异常，取首个节点标记为疑似根因
- 关键路径最多展示 3 条
- 警告提示示例：
  - "缺少链路节点数据，当前只能基于调用摘要分析"
  - "缺少任务标识，追踪退化为用户/数字员工维度"
  - "缺少调用者信息，无法准确判断影响范围"

---

#### F-08：链路时间线（降级模式）

**功能描述**: 当链路节点数据不完整（缺少时序关系信息）时，降级展示基于阶段的时间线。

**前置条件**:
- 调用详情已加载
- 该调用缺少完整链路数据但有阶段节点数据

**主要行为**:
1. 顶部展示彩色条形时间线，每个阶段按耗时占比分配宽度
2. 下方展示阶段列表：状态圆点 + 阶段名 + 摘要/模型/类型 + 耗时

**业务规则**:
- 条形宽度按该阶段耗时占总耗时的百分比计算，最小 2%
- 无阶段节点时展示提示："未读取到链路节点数据。需要上游写入链路信息后才能展示阶段级追踪。"

## 4. 交互流程

### 4.1 主流程

```
[起始点] 用户点击侧栏"调用追踪"
    ↓
[步骤1] 加载统计数据 → 渲染四个统计卡片
    ↓
[步骤2] 加载列表数据 → 渲染调用记录表格（默认第 1 页）
    ↓
[判断] 用户操作？
    ↓ [筛选]                    ↓ [展开]                   ↓ [翻页]
[步骤3a] 更新筛选条件          [步骤3b] 点击某行           [步骤3c] 点击翻页
    ↓ 重新加载列表               ↓ 加载调用详情              ↓ 加载下一页数据
    ↓ 回到步骤2                  ↓ 渲染诊断卡片             ↓ 渲染列表
                                ↓ 渲染瀑布图/时间线
                                ↓
                           [判断] 点击链路节点？
                                ↓ [是]
                           [步骤4] 高亮节点行
                                ↓ 渲染节点详情面板
                                ↓ 可折叠查看输入/输出内容
    ↓                          ↓                         ↓
[结束] 列表浏览完成          [结束] 详情查看完成         [结束] 翻页浏览完成
```

**流程说明**:
1. 页面加载时统计数据和列表数据并行获取
2. 筛选条件变更自动触发重新加载，页码重置为 1
3. 展开调用行时才加载详情数据（懒加载），避免一次拉取所有详情
4. 如果详情属于更大范围的分布式链路，会自动关联并合并展示

### 4.2 异常分支

| 异常场景 | 触发条件 | 处理方式 | 用户提示 |
|---------|---------|---------|---------|
| 列表加载失败 | 网络异常或服务不可用 | 列表置空，统计置空 | 无显式提示（静默降级） |
| 详情加载失败 | 单条调用详情请求失败 | 展示红色错误框，保留列表摘要 | "详情加载失败：{原因}。列表摘要仍可用，但无法展示完整链路。" |
| 分布式链路关联失败 | 分布式链路数据请求失败 | 降级展示当前调用详情 | 无额外提示 |
| 链路数据不完整 | 缺少时序关系信息 | 降级展示阶段时间线 | "未读取到链路节点数据。需要上游写入链路信息后才能展示阶段级追踪。" |
| 阶段数据也缺失 | 完全无链路信息 | 展示提示框 | "未读取到链路节点数据。需要上游写入链路信息后才能展示追踪。" |
| 数据质量问题 | 缺少任务标识/调用者信息 | 展示黄色警告 | "缺少任务标识，追踪退化为用户/数字员工维度"等 |

### 4.3 状态机

**调用记录状态**:

| 状态 | 含义 | 进入条件 |
|------|------|---------|
| 运行中 | 调用正在执行 | 调用发起 |
| 成功 | 调用正常完成 | LLM 正常返回结果 |
| 失败 | 调用业务逻辑失败 | 模型返回失败/超时 |
| 错误 | 调用系统级错误 | 服务异常/网络错误 |

```

[运行中] ──[调用失败]──> [失败]
[运行中] ──[系统错误]──> [错误]
[运行中] ──[风险拦截]──> [已拦截]
```

## 5. 核心概念

### 5.1 概念模型

本页面涉及三个核心概念，自上而下构成层次关系：

```
分布式链路 (Distributed Trace)
  │  一次用户请求的完整传播路径
  │  例：用户发起对话 → 多轮 LLM 调用 + 工具调用 → 返回最终回答
  │
  └── 链路节点 (Span)
       │  一次具体操作，是链路的子单元
       │  例：一次 LLM 推理、一次工具执行、一次风险审查
       │
       ├── 风险命中 (Risk Hit)
       │    调用内容被风险规则匹配到的记录
       │    例：检测到身份证号 → 命中"个人隐私信息检测"规则
       │
       └── 阶段节点 (Flow Node)
            链路节点的可视化补充，描述节点内部的阶段信息
            例：节点的输入输出内容、使用的模型、执行摘要
```

### 5.2 概念说明

**分布式链路**
| 属性 | 说明 | 示例 |
|------|------|------|
| 追踪标识 | 唯一标识 | dt-user-chat-001 |
| 根操作 | 触发链路的根操作 | 用户对话 / 代理任务 |
| 发起用户 | 触发请求的用户 | u-zhangsan |
| 数字员工 | 处理请求的数字员工 | 客服助手 |
| 会话标识 | 所属会话 | sess-001 |
| 节点数量 | 包含的链路节点数 | 6 |
| 整体状态 | 链路执行结果 | 成功 / 失败 / 已拦截 |
| 总耗时 | 从首节点到末节点的总时间 | 5200ms |
| 总 Token | 所有节点的 Token 之和 | 3850 |
| 总成本 | 所有节点的成本之和 | ¥0.107 |

**链路节点**
| 属性 | 说明 | 示例 |
|------|------|------|
| 节点标识 | 唯一标识 | span-gw-recv-001 |
| 父节点 | 所属上级节点 | span-gw-recv-001（构成调用树） |
| 操作名称 | 该节点执行的操作 | 网关接收 / LLM 推理 / 工具执行 / 风险审查 |
| 节点类型 | 节点分类 | 接收请求 / 发起调用 / 内部处理 |
| 状态 | 执行结果 | 成功 / 失败 / 已拦截 |
| 耗时 | 该节点执行时间 | 2100ms |
| 关联模型 | 使用的 LLM 模型 | claude-sonnet-4-6 |
| 输入内容 | 节点的输入数据 | 对话历史 / 工具参数 |
| 输出内容 | 节点的输出数据 | 模型回复 / 工具结果 |
| 标签 | 自定义键值对 | { tool_name: "sql_query" } |

**风险命中**
| 属性 | 说明 | 示例 |
|------|------|------|
| 规则标识 | 命中的风险规则 ID | rule-pii-001 |
| 规则名称 | 规则的展示名称 | 个人隐私信息检测 |
| 严重级别 | 风险等级 | 低 / 中 / 高 / 严重 |
| 处置动作 | 系统采取的动作 | 拦截 / 警告 / 仅记录 |
| 匹配摘要 | 匹配到的内容摘要 | "检测到身份证号模式" |

### 5.3 概念关系

| 关系 | 类型 | 说明 |
|------|------|------|
| 分布式链路 → 链路节点 | 一对多 | 一次请求链路包含多个操作节点 |
| 链路节点 → 风险命中 | 一对多 | 一次操作可能触发多条风险规则 |
| 链路节点 → 阶段节点 | 一对多 | 一次操作可能包含多个执行阶段 |
| 链路节点 → 链路节点 | 父子 | 通过父节点标识构成调用树 |

### 5.4 典型场景示例

**场景一：正常对话（多轮 LLM + 工具调用）**

用户发起对话 → 网关接收(12ms) → 风险审查(35ms) → LLM 推理第 1 轮(2.1s) → 工具执行(450ms) → LLM 推理第 2 轮(2.8s) → 网关响应(8ms)
- 总耗时：5.2s | 总 Token：3850 | 总成本：¥0.107 | 状态：成功

**场景二：风险拦截**

用户发起对话 → 网关接收(10ms) → 风险审查(28ms, 拦截) → LLM 推理(5s, 超时失败)
- 总耗时：800ms | 总 Token：280 | 总成本：¥0.01 | 状态：已拦截
- 风险命中：个人隐私信息检测 [高/拦截]

**场景三：工具重试**

代理任务 → 网关接收(8ms) → LLM 推理(1.5s) → 工具执行(3s, 连接超时失败) → 工具重试(1.2s, 成功) → LLM 推理第 2 轮(2s) → 网关响应(5ms)
- 总耗时：12s | 总 Token：2600 | 总成本：¥0.008 | 状态：成功
