## 1. 功能概述

### 1.1 功能定位

Channel 管理是管理后台的核心基础设施模块，负责统一管理平台与外部渠道应用（如 WPS、飞书、钉钉、企业微信、Matrix 等）的连接凭证和事件订阅配置。它是数字员工触达用户的消息管道入口——没有正确配置的 Channel，数字员工无法收发任何渠道消息。该模块在产品中承担「连接器」角色，是消息网关层面向运营人员的配置窗口。

### 1.2 目标用户

| 用户类型 | 特征描述 | 核心诉求 |
|---------|---------|---------|
| 平台运营管理员 | 负责渠道应用上下架、凭证轮换的运维人员 | 快速创建/验证渠道应用，确保消息通路畅通 |
| 系统管理员 | 管理整体平台配置的技术负责人 | 掌握所有渠道应用的运行状态，及时感知异常 |
| 集成对接人员 | 负责将外部平台事件对接到本系统的开发者 | 获取订阅地址、配置 Webhook 回调、验证联通性 |

### 1.3 核心价值

| 维度 | 描述 |
|------|------|
| 用户价值 | 将原本需要命令行/数据库操作的渠道配置变为可视化操作，配置效率提升 80%；联通验证一键完成，排障时间从分钟级降至秒级 |
| 业务价值 | 降低渠道接入门槛，新渠道从申请到上线从 1 天缩短至 10 分钟；统一凭证管理消除 AK/SK 散落风险 |
| 技术价值 | 建立渠道配置标准化流程，为后续多租户渠道隔离、凭证自动轮换提供基础 |

### 2.3 详细规格

#### F-01：渠道应用列表

**功能描述**: 页面主区域展示所有已配置的渠道应用，以表格形式呈现核心信息。顶部展示三个统计卡片（应用总数、已验证数、Webhook 已启用数），帮助管理者快速掌握渠道整体状态。

**前置条件**:
- 用户已登录管理后台并拥有 Channel 管理权限

**主要行为**:
1. 进入 Channel 管理页面，自动加载渠道应用列表
2. 顶部统计卡片实时计算并展示：应用总数、已验证数量、Webhook 已启用数量
3. 表格按行展示每个渠道应用的：应用信息（名称+ID）、AK/SK（脱敏显示前8位）、Webhook 状态、验证状态
4. 空列表时展示「暂无应用配置」占位文案
5. 支持刷新按钮手动重载列表

**业务规则**:
- AK/SK 在列表中必须脱敏显示，仅展示前 8 位 + 省略号
- 统计数据基于当前列表实时计算，非缓存值
- 已验证的应用显示绿色「已验证」标签，未验证显示黄色「未验证」标签

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 无任何渠道应用 | 表格区域展示空状态占位文案「暂无应用配置」 |
| 加载中 | 展示「加载中...」文案 |
| 网络异常 | 静默处理，列表置空 |

---

#### F-02：新增应用配置

**功能描述**: 通过右侧滑出的抽屉面板填写渠道应用信息，包括应用ID、名称、Access Key、Secret Key，以及可选的 Webhook 配置。保存后自动关闭抽屉并刷新列表。

**前置条件**:
- 用户已进入 Channel 管理页面

**主要行为**:
1. 点击「新增应用」按钮，右侧抽屉面板滑出
2. 填写必填字段：应用ID、应用名称、AK、SK
3. 可选开启 Webhook 并填写回调地址
4. 填完必填字段后，系统自动生成「配置订阅接口 URL」
5. 点击「保存」提交创建
6. 创建成功后自动关闭抽屉，列表刷新

**业务规则**:
- 应用ID、应用名称、AK、SK 为必填项，缺失时行内提示错误
- Webhook 启用时，Webhook 地址变为必填
- SK 输入框使用密码遮罩显示
- 保存过程中按钮显示 loading 状态，禁止重复提交
- 订阅 URL 基于应用ID + 名称 + AK + SK 自动拼装

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 必填字段为空 | 对应字段下方展示红色错误提示 |
| Webhook 已启用但地址为空 | 提示「请输入Webhook地址」 |
| 保存失败 | 表单底部展示「保存失败，请重试」 |
| 应用ID 已存在 | 统一展示保存失败提示 |

---

#### F-03：联通性验证

**功能描述**: 对已配置的渠道应用执行联通性验证，确认凭证有效且消息通路可达。验证结果以 Toast 通知反馈，同时列表中的验证状态标签实时更新。

**前置条件**:
- 渠道应用已创建

**主要行为**:
1. 点击列表操作列的联通验证图标按钮
2. 按钮进入 loading 旋转状态
3. 系统向目标渠道发送验证请求
4. 验证成功：展示成功 Toast「应用验证成功」，列表状态更新为「已验证」
5. 验证失败：展示失败 Toast（含失败原因），状态保持「未验证」

**业务规则**:
- 已验证的应用，验证按钮置灰不可点击
- 验证过程通常在 5 秒内完成
- 验证结果持久化存储，刷新页面后状态保持

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 凭证无效（AK/SK 错误） | Toast 提示验证失败原因 |
| 网络超时 | Toast 提示「应用验证异常，请稍后重试」 |
| 已验证的应用 | 验证按钮置灰，cursor: not-allowed |

---

#### F-04：订阅地址生成与复制

**功能描述**: 在新增/编辑表单中，当必填字段（应用ID、名称、AK、SK）填写完整后，系统自动生成配置订阅接口 URL。该 URL 用于回填至外部开放平台的事件订阅 HTTP 接口地址。支持一键复制。

**前置条件**:
- 新增或编辑渠道应用时
- 应用ID、名称、AK、SK 四个字段均已填写

**主要行为**:
1. 四个必填字段填写完成后，订阅 URL 区域自动展示完整地址
2. 未填完时展示引导文案「请先填写应用ID、名称、AK、SK 后自动生成」
3. 点击「复制」按钮，URL 复制到系统剪贴板
4. 复制成功 Toast 提示「订阅地址已复制到剪贴板」

**业务规则**:
- URL 自动拼装，用户不可手动编辑
- URL 基于当前页面域名动态生成

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 剪贴板权限被拒绝 | Toast 提示「复制失败，请手动选择复制」 |
| 必填字段不完整 | 不生成 URL，展示引导文案 |

---

#### F-05：编辑应用配置

**功能描述**: 通过右侧抽屉面板修改已有渠道应用的配置信息。表单预填充当前值，修改后保存即生效。

**前置条件**:
- 渠道应用已创建

**主要行为**:
1. 点击列表操作列的编辑图标按钮
2. 右侧抽屉面板滑出，表单预填充当前应用配置
3. 修改任意字段后点击「保存」
4. 保存成功后抽屉关闭，列表刷新

**业务规则**:
- 所有字段均可修改，包括应用ID和 AK/SK
- 表单校验规则同新增
- SK 字段预填充时使用密码遮罩

---

#### F-06：删除应用配置

**功能描述**: 移除不再使用的渠道应用配置。删除前需二次确认；**已绑定数字员工的渠道禁止删除**，需先解除绑定关系。删除后不可恢复。

**前置条件**:
- 渠道应用已创建

**主要行为**:
1. 点击列表操作列的删除图标按钮
2. 系统检查该渠道是否与数字员工存在绑定关系
3. **若已绑定**：弹出警告弹窗，提示无法删除，并展示绑定的数字员工信息
4. **若未绑定**：弹出确认弹窗，标题「删除应用配置」，内容「确定要删除该应用配置？此操作不可恢复。」
5. 点击「确认」执行删除
6. 删除过程中确认按钮展示 loading 状态
7. 删除成功后弹窗关闭，列表刷新

**业务规则**:
- 删除操作不可逆，必须经过二次确认
- **已绑定数字员工的渠道不允许删除**，需先在数字员工管理中解除绑定关系后才能删除
- 绑定关系由系统实时校验，确保数据一致性

**提示文案**:

| 场景 | 弹窗标题 | 弹窗内容 | 操作按钮 |
|------|---------|---------|---------|
| 渠道已绑定数字员工 | `无法删除` | `该渠道应用已绑定 {N} 个数字员工，请先解除绑定后再删除。` + 绑定的员工名称列表 | `关闭` |
| 渠道未绑定，确认删除 | `删除应用配置` | `确定要删除该应用配置？此操作不可恢复。` | `取消` / `确认` |

**绑定弹窗补充说明**:
- `{N}` 为实际绑定的数字员工数量
- 弹窗中列出绑定的数字员工名称（最多展示 5 个，超出显示「等共 N 个」）
- 「关闭」按钮仅关闭弹窗，不执行任何删除操作

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 渠道被数字员工绑定中 | 弹出警告弹窗「无法删除」，展示绑定员工信息，不执行删除 |
| 绑定关系刚好在删除前被解除 | 允许删除，进入正常确认流程 |
| 删除失败 | Toast 提示「删除失败，请重试」 |

---

#### F-07：Webhook 事件配置

**功能描述**: 控制渠道应用是否接收 Webhook 事件回调，并配置回调地址。通过开关控制启用状态。

**前置条件**:
- 新增或编辑渠道应用时

**主要行为**:
1. 在表单中切换「启用 Webhook」开关
2. 开关开启后，下方展示 Webhook 地址输入框
3. 填写 Webhook 地址
4. 保存时，若 Webhook 已启用但地址为空，校验报错

**业务规则**:
- 开关默认关闭
- Webhook 启用时地址为必填
- 地址需为合法 URL 格式

---

#### F-08：关键词搜索

**功能描述**: 在列表上方通过搜索框按应用ID或名称过滤渠道应用。

**前置条件**:
- 渠道应用列表已加载

**主要行为**:
1. 在搜索框输入关键词
2. 列表实时过滤，仅展示应用ID或名称包含关键词的记录
3. 清空搜索框恢复展示全部记录

**业务规则**:
- 搜索为前端即时过滤，不请求后端
- 搜索不区分大小写
- 支持应用ID和名称两个维度的模糊匹配

### 4.4 异常分支

| 异常场景 | 触发条件 | 处理方式 | 用户提示 |
|---------|---------|---------|---------|
| 网络超时 | 请求超过 5 秒 | Toast 通知 | 「应用验证异常，请稍后重试」 |
| 凭证无效 | AK/SK 不匹配 | Toast 通知 | 含具体的失败原因 |
| 保存失败 | 创建/更新失败 | 行内错误提示 | 「保存失败，请重试」 |
| 剪贴板拒绝 | 浏览器剪贴板权限 | Toast 通知 | 「复制失败，请手动选择复制」 |
| 数据为空 | 列表无数据 | 空状态占位 | 「暂无应用配置」 |
| 表单校验失败 | 必填项为空/格式错误 | 行内红色提示 | 如「请输入应用ID」 |
| 渠道被数字员工绑定 | 渠道已绑定数字员工 | 警告弹窗阻止删除 | 「该渠道应用已绑定 N 个数字员工，请先解除绑定后再删除。」 |

### 4.5 状态机

| 对象 | 状态列表 | 状态转换触发条件 |
|------|---------|---------------|
| 渠道应用验证状态 | 未验证 → 已验证 | 联通验证成功 |
| 渠道应用绑定状态 | 未绑定 → 已绑定 → 已解除 | 绑定数字员工 / 解除全部绑定 |
| 渠道应用生命周期 | 不存在 → 已创建 → 已删除 | 创建 / 删除操作 |

**状态转换图**:
```
[未验证] --[联通验证成功]--> [已验证]
[已验证] --[编辑凭证后]--> [未验证]（建议凭证变更后重置）

[不存在] --[创建]--> [已创建（未验证、未绑定）]
[已创建] --[绑定数字员工]--> [已绑定]
[已绑定] --[解除所有绑定]--> [未绑定]
[未绑定] --[删除]--> [不存在]（不可逆）
[已绑定] --[删除]--> ❌ 拒绝，提示先解除绑定
```

## 5. 数据模型

### 5.1 渠道应用字段说明

| 字段名 | 必填 | 说明 | 约束 |
|--------|------|------|------|
| 应用ID | 是 | 外部平台分配的应用ID | 创建时必填，全局唯一 |
| 应用名称 | 是 | 应用显示名称 | 创建时必填 |
| Access Key (AK) | 是 | 渠道平台凭证 Key | 创建时必填，列表中脱敏展示 |
| Secret Key (SK) | 是 | 渠道平台凭证 Secret | 创建时必填，列表中脱敏展示，输入时密码遮罩 |
| Webhook 地址 | 否 | 事件回调地址 | Webhook 启用时必填，需为合法 URL |
| Webhook 启用 | 否 | 是否接收事件推送 | 默认关闭 |
| 验证状态 | 否 | 联通验证是否通过 | 默认未验证，验证成功后置为已验证 |
| 绑定状态 | 否 | 是否绑定了数字员工 | 由系统根据绑定关系自动计算 |

### 5.2 实体关系

```
[Channel] 1 --- N [DigitalEmployee]
    |
    1
    |
[WebhookEvent]
```

**关系说明**:
| 关系 | 类型 | 说明 |
|------|------|------|
| Channel → DigitalEmployee | 一对多 | 一个渠道应用可被多个数字员工绑定，作为消息收发通道；**已绑定的渠道禁止删除** |
| Channel → WebhookEvent | 一对多 | 启用 Webhook 的渠道可接收多条事件回调 |
