## 1. 功能概述

### 1.1 功能定位

数字员工管理是管理后台的核心模块，承载数字员工（AI Agent 实例）的全生命周期管理。该模块提供**业务视角**和**运维视角**双视图，支持组织级数字员工的创建、配置、监控、版本管理和评测闭环，是连接 Channel 渠道、工具技能、评测基准的关键枢纽。

> ⚠️ **本期范围约束**：本期核心聚焦**组织级数字员工创建全链路**，暂不支持以下能力：
> - ① 个人级数字员工创建（个人级实例仅列表展示，不支持创建入口）
> - ② 创建向导中的评测配置步骤（向导跳过评测步骤，创建后可在详情/编辑中补关联）
> - ③ 计算资源、模型和预算配置（资源抽屉本期不开放）
> - ④ 契约规则和成长档案（主题抽屉本期不开放）

### 1.2 目标用户

| 用户类型 | 特征描述 | 核心诉求 |
|---------|---------|---------|
| 运营管理员 | 负责数字员工日常运营、配置调整 | 快速创建和配置数字员工，批量管理实例状态 |
| 运维工程师 | 关注系统稳定性和资源分配 | 实时监控实例运行状态，快速处置异常和节点漂移 |
| 业务负责人 | 关注数字员工服务质量和效果 | 查看评测结果，了解数字员工服务范围和契约边界 |

### 1.3 核心价值

| 维度 | 描述 |
|------|------|
| 用户价值 | 将组织级数字员工创建流程从"散点配置"整合为四步向导，减少配置遗漏 60%以上 |
| 业务价值 | 双视角设计让运维和业务人员各取所需，减少跨角色沟通成本；评测闭环让上线质量可度量 |
| 技术价值 | 统一实例数据模型，版本管理可追溯，为后续资源配额和契约管理奠定基础 |

### 2.3 详细规格

#### F-E01：员工列表

**功能描述**: 以表格形式展示当前租户下的所有数字员工实例，提供业务和运维两种视角切换，并支持多维度筛选。

**前置条件**:
- 用户已登录管理后台并进入"数字员工"模块
- 当前租户下存在至少一个数字员工实例（空状态有兜底展示）

**主要行为**:
1. 页面加载时自动拉取员工列表
2. 默认展示业务视角，可切换至运维视角
3. 运维视角下可切换 Agent 类型（全部/OpenClaw/Harness）和集群类型（K8s/CubeSandbox）
4. 业务视角筛选栏支持：关键词搜索、状态筛选、固定会话ID、部门、岗位、实例类型
5. 运维视角筛选栏额外支持：Node 节点、Pod 名称
6. 业务视角表格列：名称、状态、工号、部门/岗位、类型、操作
7. 运维视角表格列：名称、状态、Pod、节点、重启次数、操作
8. CubeSandbox 集群视图列：Instance ID、Agent ID、状态模式、心跳、健康状态、Runtime Template、Agent Revision
9. 点击"查看"打开详情抽屉，点击"编辑"打开编辑抽屉，点击"删除"弹出确认弹窗
10. 运维视角操作按钮：启动、停止、容器重启、漂移

**业务规则**:
- 列表按创建时间倒序排列
- 状态映射：active/running → 绿色，paused/provisioning/pending → 黄色/蓝色，inactive → 灰色，error/failed → 红色
- 部门名称自动翻译为中文（ops→运营、finance→财务等）
- 岗位名称自动翻译为中文（operator→操作员、analyst→分析师等）
- 实例类型标签：组织级（蓝色）、个人级（紫色）
- 删除和停止操作使用危险样式确认弹窗
- 运维视角下节点显示健康状态图标（✓/⚠/✕）

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 列表为空 | 表格展示"暂无员工数据"占位行 |
| 加载中 | 展示"加载中..."居中提示 |
| 列表接口异常 | 静默降级为空列表，不阻断页面 |
| 筛选无结果 | 表格展示"暂无员工数据"占位行 |
| 关键词搜索 | 同时匹配 ID、名称、显示名称、Matrix Room ID |

#### F-E02：创建向导

**功能描述**: 通过三步向导流程创建组织级数字员工，依次配置基础信息、能力配置和初始版本，降低配置遗漏风险。

> ⚠️ **本期暂不支持评测配置步骤**：向导中跳过"评测配置"步骤，数字员工创建后可在详情抽屉或编辑抽屉中补关联评测集。评测配置步骤的 UI 和逻辑保留，后续迭代开放时无需重新开发。

**前置条件**:
- 用户点击页头"创建数字员工"按钮
- Channel 管理、工具管理、技能管理模块中已有可用数据

**主要行为**:
1. **步骤一 — 基础信息**：填写员工名称（必填）、归属部门、负责人、绑定 Channel 应用（必填）、Agent Runtime（OpenClaw/Harness）、默认模型、描述、System Prompt（必填，支持一键生成）
2. **步骤二 — 能力配置**：从工具管理中选择可用工具（多选），从技能管理中选择可用 Skill（多选）
3. ~~**步骤三 — 评测配置**（可选）：选择评测集，开关"创建后立即运行基线评测"~~ — ⚠️ 本期暂不支持，向导中跳过此步骤
4. **步骤三 — 初始版本**：填写初始版本号（必填，默认 v0.1.0）、版本说明、是否创建后立即发布
5. 每步有"下一步"按钮，必填项未完成时按钮禁用
6. 已完成的步骤可点击回退修改
7. 最后一步展示配置摘要，点击"创建初始版本"提交

**业务规则**:
- 员工名称不能为空
- 必须绑定 Channel 应用（选择应用后自动填充 appId/channelName）
- System Prompt 不能为空，支持根据基础信息自动生成模板
- 自动生成 Prompt 逻辑：基于名称、部门、描述、模型、Runtime 组装行为规范模板
- 工具列表仅展示已启用的工具定义
- 评测集列表排除已归档的评测集
- 创建成功后若有未完成的后续配置，弹出 info 提示
- 创建时若勾选基线评测，成功提示中说明评测已启动

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| Channel 应用列表为空 | 展示空状态占位，提示"请先在 Channel 管理中新增应用配置" |
| 工具/Skill 列表为空 | 展示空状态占位，提示前往对应模块配置 |
| ~~评测集列表为空~~ | ~~展示空状态占位，提示前往评测基准模块创建~~ — ⚠️ 本期跳过评测步骤 |
| 远程数据加载失败 | 展示错误状态和重试按钮，不使用本地假数据 |
| 创建接口失败 | 表单内展示错误信息，不关闭弹窗 |

#### F-E03：详情抽屉

**功能描述**: 右侧抽屉展示数字员工的完整信息，分为可折叠的四个区块：基础信息、能力配置、评测配置（⚠️ 本期暂不支持）、版本管理。

**前置条件**:
- 用户在业务视角列表中点击某员工的"查看"按钮

**主要行为**:
1. 打开抽屉时拉取完整详情（列表 API 不含 profile/versions）
2. **基础信息区**（默认展开）：展示员工名称、归属部门、负责人、绑定 Channel 应用、Agent Runtime、默认模型、描述
3. **能力配置区**（默认展开）：展示已选工具列表、已选 Skill 列表、System Prompt（折叠，点击展开）
4. **评测配置区**（默认折叠）：展示评测集名称、基线评测开关状态、最近评测状态和时间、跳转评测看板按钮（⚠️ 本期暂不支持）
5. **版本管理区**（默认折叠）：展示当前版本号、发布状态、版本说明、版本历史列表（每条可发布）
6. 顶部操作栏：编辑按钮（跳转编辑抽屉）、评测记录按钮（跳转评测看板）
7. 底部提供"查看原始 JSON"开关

**业务规则**:
- 抽屉宽度 560px
- 工具和 Skill 以标签列表形式展示
- 版本状态：草稿（灰色）、审核中（黄色）、已发布（绿色）
- 未发布的版本可点击"发布"按钮
- 原始 JSON 展示完整数据结构，用于调试

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 详情拉取失败 | 降级展示列表中的基础数据 |
| 无版本记录 | 版本号展示"未记录"，版本历史不展示 |
| 发布操作失败 | 抽屉内展示红色错误提示 |

#### F-E04：编辑抽屉

**功能描述**: 右侧抽屉以四个 Tab 形式编辑数字员工的各项配置，支持独立保存。

**前置条件**:
- 用户在业务视角列表中点击某员工的"编辑"按钮，或在详情抽屉中点击"编辑"

**主要行为**:
1. 打开抽屉时拉取完整详情，填充表单
2. 顶部展示只读固定信息：实例ID、租户、绑定 Channel 应用、固定会话ID
3. **基础信息 Tab**：编辑员工名称（必填）、归属部门、负责人、Agent Runtime、默认模型、运行状态、实例类型、描述、System Prompt（必填，支持一键生成）
4. **能力配置 Tab**：展示已绑定的工具和 Skill 标签列表（只读查看）
5. **评测配置 Tab**：选择评测集（下拉）、切换"保存后运行基线评测"（⚠️ 本期暂不支持）
6. **版本管理 Tab**：展示当前版本和版本历史，可创建新版本（输入版本号和说明），历史版本可发布
7. 点击"保存数字员工"统一提交所有 Tab 的修改

**业务规则**:
- 员工名称和 System Prompt 为必填项
- 保存时若填写了新版本号，自动追加到版本历史（状态为草稿）
- 保存成功后刷新列表并关闭抽屉
- 操作反馈以行内消息形式展示（绿色成功/红色失败）

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 必填项为空 | 保存按钮点击后行内提示"请输入员工名称"/"请填写 System Prompt" |
| 保存失败 | 展示红色错误提示，不关闭抽屉 |
| 评测集下拉为空 | 展示当前绑定名称的只读文本 |

#### F-E05：运维操作

**功能描述**: 在运维视角下对数字员工实例执行生命周期操作和节点漂移。

**前置条件**:
- 用户切换至运维视角
- 目标实例处于可操作状态

**主要行为**:
1. 每行展示操作按钮：启动、停止、容器重启、漂移
2. 点击操作按钮弹出确认弹窗（二次确认）
3. 删除和停止操作使用危险样式确认弹窗
4. **节点漂移**：点击"漂移"打开漂移弹窗，展示当前实例名、当前节点、目标节点选择（下拉，排除当前节点，标注节点健康状态和实例数），确认后执行漂移
5. 操作执行期间按钮显示加载状态

**业务规则**:
- 启动/停止/容器重启/删除 均需二次确认
- 漂移目标节点下拉中标注节点状态：健康（无标注）/告警（标注"告警"）/异常（标注"异常"）
- 漂移目标节点展示该节点上的实例数量
- 漂移过程中实例会短暂重启（弹窗内有提示文案）
- 操作结果通过 Toast 通知反馈

**边界条件**:
| 边界场景 | 处理方式 |
|---------|---------|
| 操作失败 | Toast 提示失败原因 |
| 无可用漂移节点 | 目标节点下拉为空 |
| 重复操作 | 操作期间按钮禁用，防止重复提交 |

## 4. 交互流程

### 4.1 主流程：创建数字员工

```
[起始点] 用户点击"创建数字员工"按钮
    ↓
[步骤1] 填写基础信息（名称、部门、Channel、Runtime、模型、Prompt）
    ↓ [必填完成]           ↓ [必填未完成]
[步骤2] 选择能力配置       [下一步]按钮禁用
    ↓
~~[步骤3] 配置评测（可选）~~ — ⚠️ 本期跳过
    ↓
[步骤3] 填写版本信息 + 确认摘要
    ↓
[判断] 提交创建
    ↓ [成功]               ↓ [失败]
[结束] 关闭弹窗             [结束] 展示错误，保留表单
    刷新列表
    Toast 提示结果
```

**流程说明**:
1. 步骤指示器实时显示当前进度，已完成步骤可点击回退
2. 基础信息中 System Prompt 支持一键生成，基于已填写信息自动组装
3. 能力配置步骤数据为空时展示空状态引导
4. 版本步骤展示前几步的配置摘要，便于最终确认
5. ⚠️ 本期跳过评测配置步骤，创建后可在详情/编辑抽屉中补关联评测集

### 4.2 主流程：运维操作

```
[起始点] 用户切换至运维视角
    ↓
[步骤1] 定位目标实例（筛选/搜索）
    ↓
[步骤2] 选择操作（启动/停止/重启/漂移/删除）
    ↓
[判断] 操作类型
    ↓ [漂移]                ↓ [其他操作]
[步骤3a] 打开漂移弹窗       [步骤3b] 弹出确认弹窗
    选择目标节点
    ↓                       ↓
[步骤4] 确认执行             确认执行
    ↓                       ↓
[结束] Toast 反馈结果        Toast 反馈结果
    刷新列表                 刷新列表
```

### 4.3 主流程：编辑数字员工

```
[起始点] 用户点击"编辑"按钮
    ↓
[步骤1] 打开编辑抽屉，拉取完整详情
    ↓
[步骤2] 切换 Tab 修改配置（基础/能力/评测/版本）
    ↓
[步骤3] 点击"保存数字员工"
    ↓ [校验通过]            ↓ [校验失败]
[步骤4] 提交保存             行内提示错误
    ↓
[结束] Toast 反馈 + 刷新列表 + 关闭抽屉
```

### 4.4 异常分支

| 异常场景 | 触发条件 | 处理方式 | 用户提示 |
|---------|---------|---------|---------|
| 列表加载失败 | 后端服务不可用 | 静默降级为空列表 | 无（页面展示空状态） |
| 详情拉取失败 | 单条详情接口异常 | 降级展示列表基础数据 | 无 |
| 创建失败 | 必填校验未通过/接口异常 | 表单内展示错误 | "请输入员工名称" / "创建失败" |
| 保存失败 | 接口异常 | 行内红色提示 | "保存失败：xxx" |
| 运维操作失败 | 实例状态不匹配/接口异常 | Toast 提示 | "节点漂移失败：xxx" |
| Channel/工具/Skill 加载失败 | 远程服务异常 | 展示错误状态 + 重试按钮 | "Channel 应用加载失败" |
| 发布版本失败 | 接口异常 | 行内红色提示 | "发布失败：xxx" |
| 删除操作 | 用户点击删除 | 危险样式确认弹窗 | "确定要执行删除操作吗？" |

## 5. 数据模型

> 本章从产品视角描述核心实体及其关系，不含技术实现细节。

### 5.1 核心实体：数字员工（Employee）

数字员工是本模块的核心实体，代表一个运行在平台上的 AI Agent 实例。

**关键字段分组**:

| 分组 | 字段 | 说明 | 本期状态 |
|------|------|------|---------|
| **身份** | 员工名称、显示名称、工号 | 唯一标识和展示 | ✅ 支持 |
| **归属** | 租户、部门、岗位、负责人 | 组织关系 | ✅ 支持 |
| **类型** | 实例范围（组织级/个人级）、Agent 运行时（OpenClaw/Harness） | 分类维度 | ✅ 支持（仅创建组织级） |
| **渠道** | Channel 应用、固定会话 ID | 消息接入 | ✅ 支持 |
| **能力** | 已绑定工具、已绑定 Skill、System Prompt | 能力边界 | ✅ 支持 |
| **运行** | 运行状态、Pod 名称、节点名称、重启次数 | 运维监控 | ✅ 支持 |
| **评测** | 评测集、基线评测开关、最近评测状态 | 质量度量 | ✅ 详情/编辑支持，⚠️创建向导暂不支持 |
| **版本** | 当前版本号、版本历史、发布状态 | 变更追溯 | ✅ 支持 |
| **资源** | CPU/内存/GPU、模型/并发/降级、预算、存储 | 资源配额 | ⚠️ 本期暂不支持独立配置 |
| **契约** | Allow/Deny 列表、KPI、审批策略 | 权限边界 | ⚠️ 本期暂不支持 |
| **成长** | 能力标签、知识领域、技能认证、成长路径 | 能力发展 | ⚠️ 本期暂不支持 |

### 5.2 实例状态

| 状态 | 含义 | 可执行操作 |
|------|------|-----------|
| provisioning | 创建中 | 等待 |
| pending | 待启动 | 启动 |
| active / running | 运行中 | 暂停、停止 |
| paused | 暂停 | 启动、停止 |
| inactive | 停用 | 启动 |
| error / failed | 异常 | 重启、停止 |

### 5.3 版本状态

| 状态 | 含义 | 可执行操作 |
|------|------|-----------|
| draft | 草稿 | 发布、提交审核 |
| review | 审核中 | 审核通过、审核拒绝 |
| published | 已发布 | — |

### 5.4 实体关系

```
[Tenant] 1 ─── N [Employee]
                  │
    ┌─────────────┼─────────────┐
    │             │             │
    N             N             N
    │             │             │
[Channel]    [ToolDef]      [Skill]
    │             │             │
  1:N          N:N           N:N

[EvalSuite] 1 ─── N [Employee]    (评测集关联)
[Version]   1 ─── N (嵌套在 Employee 内)
```

| 关系 | 类型 | 说明 |
|------|------|------|
| 租户 → 员工 | 一对多 | 一个租户可拥有多个数字员工实例 |
| Channel → 员工 | 一对多 | 一个 Channel 应用可绑定多个数字员工 |
| 评测集 → 员工 | 一对多 | 一个评测集可关联多个数字员工 |
| 工具 ↔ 员工 | 多对多 | 工具可被多个员工使用，员工可使用多个工具 |
| Skill ↔ 员工 | 多对多 | Skill 可被多个员工使用，员工可拥有多个 Skill |
| 员工 → 版本 | 一对多 | 一个员工可拥有多个版本记录 |
