## 1. 功能概述

### 1.1 功能定位

权限管理是管理后台的「系统」分组下的核心模块，负责管理后台的用户生命周期（创建、查看、编辑、删除）与认证方式管理。它是系统安全的第一道防线——确保只有经过授权的人员才能访问管理后台，且操作范围受角色约束。

**认证体系设计**：HMR 平台独立维护一套账号体系，用于认证和唯一标识。同时支持两种登录方式：
1. **本地密码登录** — HMR 原生账号体系，用户名 + 密码
2. **企业 SSO（OIDC）登录** — 对接企业身份源（IdP，如企业自有 OAuth/OIDC 认证中心），通过 OIDC 协议完成鉴权，以邮箱字段做用户关联

两种方式共存，用户可选择任一方式登录。SSO 登录时，系统根据 OIDC 返回的邮箱匹配本地用户；**只有被赋予管理员角色的用户才允许登录平台**，SSO 鉴权通过但未在 HMR 被赋予角色的成员不允许登录。

**角色体系**：本期简化为「管理员」单一默认角色，不实现细粒度权限矩阵与 RBAC 全量逻辑。用户创建时角色默认选中管理员，无需手动选择。

**企业 SSO 对接方式**：本期采用线下对接方案——OIDC 参数（Issuer、ClientID、ClientSecret 等）通过环境变量配置，由运维人员部署时设置，不提供管理后台在线配置入口。企业可接入自有 IdP（如 Entra ID、Okta、Keycloak、自建 OIDC Provider 等）。

### 1.2 目标用户

| 用户类型 | 特征描述 | 核心诉求 |
|---------|---------|---------|
| 平台管理员 | 拥有最高权限，负责平台级用户与租户管理 | 快速创建/管理后台用户，掌控系统访问权限 |
| 租户管理员 | 租户内最高权限，管理本租户下的员工与配置 | 管理本租户内的用户账号 |
| 企业员工（SSO 用户） | 通过企业 SSO 登录，不直接管理 HMR 平台密码 | 用企业账号直接登录，不用记忆额外密码 |
| 运维人员 | 负责系统部署与健康监控 | 配置 SSO 环境变量，查看认证服务状态 |

### 1.3 核心价值

| 维度 | 描述 |
|------|------|
| 用户价值 | 管理员 30 秒完成新用户创建；企业员工用已有企业账号直接登录，零额外记忆负担 |
| 业务价值 | 统一身份管理入口，对接企业 SSO 消除账号孤岛；角色校验确保未授权人员无法进入 |
| 技术价值 | HMR 独立账号体系保证认证与标识的自主可控；OIDC 标准协议对接，可扩展至任意 OIDC 兼容 IdP |

### 2.3 详细规格

#### F-01：用户列表

**功能描述**: 以表格形式展示所有后台用户，支持快速扫描用户状态与角色信息。每行提供编辑、删除操作入口。

**前置条件**:
- 用户已登录管理后台

**主要行为**:
1. 进入权限管理页面，默认展示「用户」Tab
2. 系统并行拉取用户列表、角色列表、认证健康状态
3. 表格按创建时间倒序排列，最新用户在最前
4. 每行展示：用户名、邮箱、角色（Badge 样式）、操作列

**业务规则**:
- 用户名唯一，不可重复
- 角色显示为 Badge，本期统一为「管理员」
- 操作列：编辑（铅笔图标）、删除（垃圾桶图标）
- 列表为空时展示空状态页，文案："暂无用户，点击新建"

**边界条件**:

| 边界场景 | 处理方式 |
|---------|---------|
| 用户列表为空 | 展示空状态页 + 「新建用户」引导按钮 |
| 网络请求失败 | 展示错误提示 + 重试按钮 |

#### F-02：创建用户

**功能描述**: 通过右侧抽屉（Drawer）表单创建新用户。本期简化为三字段 + 默认角色，降低操作门槛。

**前置条件**:
- 用户已登录管理后台

**主要行为**:
1. 点击「新建用户」按钮，右侧滑出 Drawer
2. 表单包含：用户名（必填）、邮箱（必填）、初始密码（必填）、角色（默认「管理员」，本期不可修改）
3. 填写完成后点击「创建」，提交到后端
4. 创建成功：Drawer 关闭，列表刷新，新用户出现在首行，Toast 提示"用户创建成功"
5. 创建失败：表单内展示具体错误信息（如用户名已存在）

**业务规则**:
- **用户名**：必填，2-32 位，仅允许字母、数字、下划线
- **邮箱**：必填，标准邮箱格式校验；邮箱在系统内唯一，作为 SSO 关联的匹配键
- **初始密码**：必填，最少 6 位
- **角色**：默认选中「管理员」，下拉框置灰不可切换（Tooltip 提示"后续版本支持自定义角色"）
- 提交时前端校验不通过，对应字段下方展示红色错误提示
- 提交中按钮显示 loading 状态，防止重复提交

**边界条件**:

| 边界场景 | 处理方式 |
|---------|---------|
| 用户名已存在 | 用户名字段下方提示"该用户名已被占用" |
| 邮箱已存在 | 邮箱字段下方提示"该邮箱已被占用" |
| 邮箱格式不合法 | 邮箱字段下方提示"请输入有效的邮箱地址" |
| 密码少于 6 位 | 密码字段下方提示"密码至少 6 位" |
| 提交中网络断开 | Toast 提示"网络异常，请重试"，按钮恢复可点击 |

#### F-03：编辑用户

**功能描述**: 通过右侧抽屉编辑已有用户信息，可修改角色。

**前置条件**:
- 用户已登录管理后台
- 目标用户存在

**主要行为**:
1. 在用户列表点击某行的「编辑」按钮，右侧滑出 Drawer（复用创建 Drawer，切换为编辑模式）
2. 表单预填当前用户信息
3. 可修改字段：角色
4. 不可修改字段：用户名（只读展示）、邮箱（只读展示）、密码（不可在此重置）
5. 修改后点击「保存」，提交到后端
6. 保存成功：Drawer 关闭，列表对应行更新，Toast 提示"用户信息已更新"

**业务规则**:
- **角色**：下拉选择，本期仅有「管理员」选项
- **用户名**：编辑模式下只读，不可修改
- **邮箱**：编辑模式下只读，不可修改

#### F-04：删除用户

**功能描述**: 删除用户账号，操作不可逆。删除同时清除外部身份关联记录。

**前置条件**:
- 用户已登录管理后台
- 目标用户不是当前登录用户自身

**主要行为**:
1. 在用户列表点击「删除」按钮
2. 弹出二次确认弹窗："确定要删除用户 XXX 吗？此操作不可恢复"
3. 确认后执行删除，列表移除该行，Toast 提示"用户已删除"

**业务规则**:
- 删除后用户无法登录
- 删除用户时同步清除外部身份关联记录
- 删除自己：按钮置灰

#### F-05：角色概览

**功能描述**: 权限管理页面的第二个 Tab，展示当前系统中的角色列表。本期为只读展示，不支持创建/编辑角色。

**前置条件**:
- 用户已登录管理后台

**主要行为**:
1. 切换到「角色」Tab
2. 展示角色列表：角色名、权限数、描述
3. 本期仅有「管理员」一条记录
4. 操作列为空（或展示"后续版本支持自定义角色"的 Tooltip）

**业务规则**:
- 本期角色列表不可编辑、不可新增、不可删除
- 后续迭代开放自定义角色与权限矩阵配置

#### F-06：SSO 登录入口与角色校验

**功能描述**: 在管理后台登录页增加企业 SSO 登录入口。SSO 认证通过后，系统校验用户是否被赋予管理员角色——只有被授权的管理员才能登录平台。

**前置条件**:
- OIDC 配置已完成（由运维人员线下设置），SSO 服务可用

**主要行为**:
1. 登录页在本地密码登录表单下方展示分隔线 + 「企业 SSO 登录」按钮
2. 用户点击「企业 SSO 登录」
3. 系统重定向至企业 IdP 认证端点
4. 用户在企业 IdP 完成认证
5. 企业 IdP 回调至系统，携带授权码
6. 系统换取用户信息（含邮箱）
7. **用户查找与角色校验**（核心逻辑）：
   - 按邮箱匹配本地用户
   - 未匹配：按外部身份标识匹配
   - 匹配成功 + 用户拥有管理员角色 → 登录成功，跳转管理后台首页
   - 匹配成功但用户无管理员角色 → 拒绝登录，提示"您未被授权访问该平台，请联系管理员"
   - 完全未匹配 → 拒绝登录，提示"该企业账号未关联系统用户，请联系管理员创建账号"
8. 首次 SSO 登录成功后，自动建立外部身份与本地用户的关联

**业务规则**:
- **角色是登录的硬性门槛**：无论通过本地密码还是 SSO 登录，用户必须被赋予管理员角色才能进入平台。未赋予角色的成员即使 SSO 鉴权通过也不允许登录。
- SSO 未启用时（OIDC 未配置），登录页不展示 SSO 入口按钮
- 不自动注册 SSO 用户：SSO 用户必须由管理员预先创建账号

**边界条件**:

| 边界场景 | 处理方式 |
|---------|---------|
| SSO 鉴权通过但未在 HMR 创建对应用户 | 拒绝登录，提示"未关联系统用户" |
| SSO 鉴权通过但用户无管理员角色 | 拒绝登录，提示"未被授权访问该平台" |
| SSO 未配置 | 登录页不展示 SSO 按钮 |
| 回调校验失败 | 拒绝登录，提示"登录异常，请重试" |

#### F-08：邮箱关联

**功能描述**: SSO 登录的核心关联机制——以邮箱为匹配键，将 OIDC 外部身份与 HMR 本地用户绑定。

**前置条件**:
- SSO 已启用且配置正确
- 企业 IdP 返回的用户信息中包含邮箱

**主要行为**:
1. 用户通过 SSO 完成企业 IdP 认证
2. 系统提取用户邮箱
3. 按邮箱匹配本地用户
4. 匹配成功：进入角色校验流程（F-07 步骤 7）
5. 匹配失败：按外部身份标识查找
6. 首次 SSO 登录成功后自动创建外部身份与本地用户的关联记录

**业务规则**:
- **邮箱是主要匹配键**：系统内邮箱不可重复，确保 SSO 关联的确定性
- **关联记录**：首次 SSO 登录成功后自动创建关联记录
- **邮箱变更**：若管理员修改了本地用户的邮箱，下次 SSO 登录按新邮箱匹配
- **不自动注册**：SSO 用户必须由管理员预先创建

**边界条件**:

| 边界场景 | 处理方式 |
|---------|---------|
| 企业 IdP 返回信息缺少邮箱 | SSO 登录失败，提示"SSO 返回信息缺少邮箱，无法关联" |
| 本地无匹配邮箱的用户 | 尝试按外部身份标识匹配；仍未匹配则拒绝登录 |
| 匹配到的用户无管理员角色 | 拒绝登录，提示"未被授权访问该平台" |

---

#### F-09：登录审计日志

**功能描述**: 记录每次登录事件（成功与失败），遵循 5W1H 审计要素原则，确保信息完整性，便于安全审计与风险规避。

**前置条件**:
- 审计日志基础设施已就绪

**主要行为**:
1. 每次登录事件（无论成功/失败）触发审计日志记录
2. 日志可在管理后台的「日志审计」模块中查询

**5W1H 审计要素**:

| 要素 | 字段 | 说明 | 示例 |
|------|------|------|------|
| **Who**（谁） | 用户名、用户ID | 操作人身份 | zhangsan / ID:5 |
| **When**（何时） | 登录时间 | 事件发生时间 | 2026-06-08 14:30:22 |
| **Where**（何地） | IP 地址、User-Agent | 请求来源 | 10.0.1.55 / Chrome 120 |
| **What**（做什么） | 操作类型 | 登录成功 / 登录失败 | login_success / login_failure |
| **Why**（为什么） | 失败原因 | 仅失败事件记录 | 无平台角色 / 密码错误 / 用户未找到 |
| **How**（怎么做） | 认证方式 | 通过何种方式登录 | 本地密码 / 企业SSO |

**业务规则**:
- 本地密码登录：记录成功/失败事件
- SSO 登录：记录认证成功、角色校验失败、用户未找到等事件
- 日志不可篡改、不可删除
- 保留期 90 天

**边界条件**:

| 边界场景 | 处理方式 |
|---------|---------|
| 登录接口被暴力破解 | 记录每次失败尝试（含 IP），后续可通过日志分析异常频率 |
| SSO 认证通过但角色校验失败 | 记录为登录失败，Why="无平台角色"，How="企业SSO" |

## 4. 交互流程

### 4.1 主流程：创建用户

```
[起始点] 管理员进入权限管理页面
    ↓
[步骤1] 点击「新建用户」按钮
    ↓
[步骤2] 右侧滑出 Drawer，表单为空，角色默认「管理员」
    ↓
[步骤3] 填写用户名、邮箱、初始密码
    ↓
[判断] 表单校验是否通过？
    ↓ [是]                        ↓ [否]
[步骤4a] 点击「创建」按钮      [步骤4b] 对应字段展示红色错误提示
    ↓                              ↓
[判断] 后端返回是否成功？      [步骤5b] 修正后重新提交
    ↓ [是]       ↓ [否]
[步骤5a]       [步骤5a-err]
Drawer 关闭    表单展示后端错误信息
列表刷新       按钮恢复可点击
Toast 提示成功
```

**流程说明**:
1. 角色字段默认选中「管理员」且不可切换，减少操作步骤
2. 前端校验在字段失焦时触发，提交时做全量校验
3. 创建成功后列表自动刷新，新用户出现在首行

### 4.2 主流程：SSO 登录（含角色校验）

```
[起始点] 用户在登录页点击「企业 SSO 登录」
    ↓
[步骤1] 重定向至企业 IdP 认证页面
    ↓
[步骤2] 用户在企业 IdP 完成认证
    ↓
[步骤3] 企业 IdP 回调至系统
    ↓
[判断] 回调校验是否通过？
    ↓ [是]              ↓ [否]
[步骤4]             拒绝登录
提取用户邮箱        提示"登录异常，请重试"
    ↓
[判断] 本地是否有匹配邮箱/外部身份的用户？
    ↓ [是]                    ↓ [否]
[步骤5]                    拒绝登录
检查用户状态和角色         提示"未关联系统用户，
                          请联系管理员创建账号"
    ↓
[判断] 用户拥有管理员角色？
    ↓ [是]                    ↓ [否]
[步骤6]                     拒绝登录
登录成功                    "未被授权访问该平台，
跳转管理后台首页            请联系管理员"
```

**流程说明**:
1. SSO 认证通过 ≠ 可以登录。**角色校验是独立的门槛**——必须被赋予管理员角色
2. SSO 用户必须由管理员预先创建，系统不自动注册
3. 登录成功/失败均记录审计日志

### 4.3 异常分支

| 异常场景 | 触发条件 | 处理方式 | 用户提示 |
|---------|---------|---------|---------|
| 网络超时 | 请求超时 | 展示重试按钮 | "网络异常，请重试" |
| 用户列表为空 | 系统无用户 | 展示空状态页 | "暂无用户，点击新建" |
| 表单校验失败 | 必填项为空/格式错误 | 行内错误提示 | "[字段名]不能为空" / "请输入有效的邮箱地址" |
| 用户名冲突 | 创建时用户名已存在 | 行内提示 | "该用户名已被占用" |
| 邮箱冲突 | 创建时邮箱已存在 | 行内提示 | "该邮箱已被占用" |
| 删除自己 | 删除当前登录用户 | 按钮置灰 | "不能删除当前登录账号" |
| SSO 鉴权通过但无角色 | 用户存在但无管理员角色 | 拒绝登录 | "您未被授权访问该平台，请联系管理员" |
| SSO 用户未在本地创建 | 邮箱/外部身份均无匹配 | 拒绝登录 | "该企业账号未关联系统用户，请联系管理员创建账号" |
| SSO 回调校验失败 | 回调参数异常或过期 | 拒绝登录 | "登录异常，请重试" |
| SSO 返回信息缺少邮箱 | 企业 IdP 返回的用户信息无邮箱 | 拒绝登录 | "SSO 返回信息缺少邮箱，无法关联" |

### 4.4 状态机

| 对象 | 状态列表 | 状态转换触发条件 |
|------|---------|---------------|
| 用户账号 | 正常 → 已删除 | 管理员删除（终态，不可逆） |
| 外部身份关联 | 未关联 → 已关联 | 首次 SSO 登录成功自动创建 |
| 外部身份关联 | 已关联 → 已解除 | 删除用户时级联删除 |

**状态转换图**:
```
[正常] --[删除操作]--> [已删除]（终态）

[未关联] --[首次SSO登录成功]--> [已关联]
[已关联] --[删除用户]--> [已解除]（终态）
```

> 注：禁用/启用状态为后续扩展功能，本期不实现。
