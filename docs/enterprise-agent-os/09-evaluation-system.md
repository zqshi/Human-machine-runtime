# 09 — 考核体系：人与 Agent 的双轨评价

## 核心原则

人和 Agent 必须用不同的尺子。人的价值不在执行，在判断。

---

## Agent 考核：纯客观绩效

| 指标 | 说明 |
|------|------|
| 任务完成率 | 接了多少、完成了多少 |
| 验收通过率 | 完成的里面，有多少一次通过验收标准 |
| 执行效率 | 时间、Token 消耗、API 调用成本 |
| 异常上报准确率 | 该报的报了没有、不该报的是不是乱报 |
| 纠偏后重复出错率 | 人纠偏一次后，同类问题是否还犯 |

### 考核结果的应用

Agent 的考核结果直接用于**编排层的路由决策**：

- 表现差的 Agent → 降权或替换
- 表现好的 Agent → 获得更多任务、更高自主权
- 本质上是"哪个好用用哪个"的自动化版本

---

## 人的考核：判断质量 + 系统贡献

传统 KPI（完成了多少任务）完全不适用于 Agent 时代的关键人才。

| 指标 | 说明 |
|------|------|
| 判断准确率 | 做出的方向性判断，事后验证对了多少 |
| 信号识别及时性 | 发现问题比系统自动检测早多少 |
| 纠偏有效性 | 人介入纠偏后，下游执行效果改善了多少 |
| 知识沉淀贡献 | 有多少判断逻辑被系统学习并复用 |
| Agent 编排能力 | 任务拆解的合理性、Agent 选择的准确性 |

### 人才标准的根本转变

在这套体系里，优秀的人才不是"自己干得好"的人，而是**"能让 Agent 集群干得好"的人**。

这意味着：
- 招聘标准变了：不看执行力，看判断力和系统思维
- 晋升标准变了：不看产出量，看决策质量和知识沉淀贡献
- 培养方向变了：不是教人做事，是教人做判断和编排 Agent
