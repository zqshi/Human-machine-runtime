# 11 — 数据接入层：连接器集合

## 定位

不是重新造一个数据中台，是一组**轻量级适配器**，从企业已有工具中被动抽取信号。

## 核心组件

| 组件 | 职责 | 技术类比 |
|------|------|---------|
| 连接器（Connector） | 对接每个外部系统的 API | Airbyte / Fivetran 的连接器模型 |
| 事件流 | 实时接收和分发事件 | Kafka / Redis Streams |
| 数据标准化 | 把不同来源的数据转成统一格式 | ETL pipeline |

## 关键设计决策

连接器必须是**插件式**的：

- 企业用飞书 → 装飞书连接器
- 企业用钉钉 → 装钉钉连接器
- 核心系统不绑定任何特定工具

## 构建策略

**复用开源（Airbyte 等）+ 自建适配。** 连接器是脏活累活，不是核心壁垒，不值得从零自建。
