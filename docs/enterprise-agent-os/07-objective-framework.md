# 07 — 目标管理体系：从 OKR 到分层目标

## 传统 OKR 在 Agent 时代的局限

OKR 的本质是两件事：**对齐**（Alignment）和**可衡量**（Measurability）。这两个需求在 Agent 时代不但没消失，反而更强烈——Agent 比人更需要明确的目标定义和验收标准。

但传统 OKR 有三个致命问题：

### 问题一：节奏太慢

传统 OKR 季度制。Agent 执行速度以天甚至小时为单位。用季度 OKR 管高速迭代的 Agent 集群，等季度复盘时 Agent 可能已跑偏两个月。

### 问题二：为人的动机设计

OKR 的 O（Objective）设计得有感召力，是为了激励人。Agent 不需要被激励，它需要**无歧义的验收条件**。"成为东南亚最受信赖的品牌"对人是好的 O，对 Agent 是废话。

### 问题三：没有处理人机混合分工

传统 OKR 假设执行者都是人。当执行者一半是人一半是 Agent 时，同一个 KR 下面人和 Agent 的职责怎么划分、怎么分别评价，OKR 框架本身不回答。

## 分层目标体系

```
┌─────────────────────────────────────────────────────────────────┐
│  L0 战略目标（Strategic Objective）                              │
│  周期：半年 / 年                                                 │
│  制定者：老板                                                    │
│  格式：方向 + 核心约束                                           │
│  例："东南亚营收翻倍，利润率不低于 15%"                            │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────────┐
│  L1 判断目标（Judgment Objective）                               │
│  周期：月度                                                      │
│  承担者：关键人才                                                 │
│  格式：需要持续回答的关键问题                                      │
│  例："泰国市场降价策略是否可持续？需要在本月内给出判断"              │
│  考核方式：判断质量（事后验证准确率）+ 信号识别及时性                │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────────┐
│  L2 执行目标（Execution Objective）                              │
│  周期：周 / 天 / 实时                                            │
│  承担者：Agent                                                   │
│  格式：任务契约（目标 + 输入 + 验收标准 + 约束 + 上报条件）         │
│  例："完成泰国市场 Q1 竞品价格数据采集，覆盖 TOP20 SKU，            │
│       准确率 >95%，3 天内交付"                                    │
│  考核方式：纯绩效指标（完成率、准确率、速度、成本）                  │
└─────────────────────────────────────────────────────────────────┘
```

### 核心设计原则

人和 Agent 被放在**不同层级、不同节奏、不同评价标准**下管理。OKR 的对齐逻辑保留，但实现方式从"人管人"进化为"人管 Agent + 系统管 Agent + 系统辅助管人"。
