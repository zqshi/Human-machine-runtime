# 03 — 冷启动策略：战略解码与信息采集

## 核心挑战

战略解码不是一个算法问题，是一个**信息密度问题**。系统要做出靠谱的任务拆解，至少需要三类输入：

| 输入类型 | 含义 | 示例 |
|---------|------|------|
| 战略意图 | 老板想去哪（方向） | "明年东南亚营收翻倍" |
| 组织现状 | 当前有什么资源、能力、约束（起点） | 团队规模、技术栈、产能利用率 |
| 环境认知 | 市场、竞争、客户的真实状态（地形） | 竞品动态、客户反馈、政策变化 |

**冷启动困境**：系统刚上线时，这三类信息要么不在系统里，要么散落的、矛盾的、过时的。

---

## 三阶段冷启动路径

### 第一阶段：旁听者——不做决策，只做结构化

**不要让系统做解码，让系统"旁听"解码。**

大多数企业已经有战略解码动作，只是用的是 OKR 表格、战略会议、周会纪要这些载体。系统第一步只做一件事：

- 接入现有文档（战略 PPT、OKR 表、会议纪要、周报）
- 接入现有系统数据（CRM、ERP、项目管理工具、代码仓库、IM 聊天记录）
- **不做决策，只做结构化**

```
输入：散落的文档、对话、数据
输出：结构化的组织现状快照
      ├── 当前在做什么（事实，不是计划）
      ├── 谁在做什么（人和事的映射）
      ├── 哪些事卡住了、为什么卡住
      └── 哪些资源是闲置的
```

**核心价值**：帮老板第一次看到组织的真实状态。大多数老板对自己组织的理解是滞后且失真的——信息在层层汇报中被过滤和美化过了。

### 第二阶段：提问者——苏格拉底式引导

当系统积累了足够的组织现状数据后，升级为**向人提出正确问题**的角色。

传统战略解码的问题：
- 没人敢问老板"这个方向的底层假设是什么"
- 中层各自拆解后互相矛盾，但没人看得到全局

系统在这个阶段的角色——苏格拉底式提问者：

```
老板说："明年东南亚营收翻倍"

系统不直接拆任务，而是提问：
├── 翻倍的基准是含越南市场还是不含？（发现定义模糊）
├── 当前东南亚团队产能利用率已 92%，翻倍意味着必须扩团队或提效，优先哪个？（暴露资源约束）
├── 竞品 X 上季度在泰国降价 30%，翻倍目标是否已考虑这个变量？（补充环境信息）
└── 营收翻倍和利润率目标之间存在冲突（客单价趋势下行），如何取舍？（暴露目标冲突）
```

**这些问题不是 AI 凭空生成的，是从第一阶段积累的组织数据和外部数据中推导出来的。**

### 第三阶段：参与者——给出标注不确定性的方案草稿

经历几轮"旁听→提问→人类修正"循环后，系统积累了：
- 这个组织的拆解习惯和逻辑
- 哪些假设最终被验证、哪些被推翻
- 不同类型任务的实际执行周期和成功率

此时系统才有资格给出拆解建议：

```
系统输出：
├── 基于战略意图和组织现状，建议拆解为以下 4 条执行路径
├── 路径 A 置信度高（有历史数据支撑）
├── 路径 B 置信度中（依赖未验证假设，标记需要人工判断）
├── 路径 C 和 D 互斥，需要老板做取舍
└── 以下 3 个关键信息缺失，建议在启动前补齐
```

**系统永远不应该输出"完美方案"，而是输出"标注了不确定性的方案草稿"。** 确定性给人信心，但标注不确定性才给人做出好决策的基础。

## 一线信息采集：零额外负担原则

**如果让一线专门为系统填表、写报告，这个系统必死。**

正确做法：从一线已有工作流中**被动采集**：

| 信息来源 | 采集方式 | 能提取什么 |
|---------|---------|-----------|
| IM 消息（飞书/钉钉/Slack） | 语义分析 | 卡点、情绪信号、跨部门协作摩擦 |
| 代码提交 / PR | 自动分析 | 实际开发进度 vs 计划、技术债务趋势 |
| CRM / 客户工单 | 结构化提取 | 客户真实需求、流失信号 |
| 会议录音 | 转写 + 摘要 | 决策点、分歧、未解决问题 |
| 任务管理工具 | 状态同步 | 实际完成率、延期模式 |

一线员工不需要知道系统在"采集信息"，他们只是在正常工作。系统的智能在于从日常工作的副产品中提取战略级信号。

### 隐私与合规边界

哪些数据可以采集、如何脱敏、谁能看到什么粒度——这本身就是产品设计的关键决策点，不能回避。

---

## 冷启动总结

核心逻辑：**先寄生、后共生、最后成为基础设施**

| 阶段 | 系统角色 | 对组织的要求 | 产出 |
|------|---------|-------------|------|
| 寄生 | 旁听者 | 零——不改变任何人的工作方式 | 组织现状快照 |
| 共生 | 提问者 | 低——回答系统提出的问题 | 高质量的战略拆解 |
| 基础设施 | 参与者 | 中——信任系统的建议并给予反馈 | 完整的战略执行闭环 |

**最怕反过来——上来就要求所有人改变工作方式来适应系统。那不是冷启动，那是找死。**
