# 01 — 组织模型分析：Agent 时代的三层结构

## 起点：Agent 对企业组织的冲击

大模型推理能力不断提升，Agent 框架不断成熟，Harness 等工具不断推出。未来企业内会有大量 Agent 支撑工作。这对传统组织结构产生根本性冲击。

## 初始假设（原始讨论）

- 老板是战略掌控者，完成战略定位后，下面都是拆解的工作
- Agent 足以高质量完成拆解后的执行工作
- HR 辅助做战略解码，根据执行路径所需人才进行岗位编制确定
- 招聘上来的人，主要工作是对执行方向的纠偏
- 只有完成战略目标后才涉及更多方向的探索

## 对初始假设的修正

### 问题一：战略→拆解→执行 不是线性过程

现实中战略不是老板拍完脑袋就定死的。大量有效战略是从一线执行反馈中涌现出来的（Mintzberg 的 emergent strategy）。一线全是 Agent 时，谁给老板提供涌现信号？Agent 能执行任务，但它不会主动发现"客户其实根本不在乎我们主推的功能，他们真正痛的是另一个点"。

**缺了中间层的感知和判断能力，老板就是个瞎子。**

### 问题二："纠偏" 是最难的部分，不是轻活儿

纠偏需要：
- **对模糊信号的识别**：市场风向变了，但数据还没体现出来
- **跨域的 contextual judgment**：技术可行但商业上不成立
- **政治判断和利益协调**：这个方向动了谁的蛋糕

这些恰恰是当前 Agent 最弱的能力。

### 问题三：HR 角色被过度工具化

在 Agent 密集型组织里，HR 面临的真正挑战是：
- **人机协作模式设计**：什么工作给人、什么给 Agent、怎么衔接
- **组织能力重新定义**：执行层被 Agent 占据后，"核心能力"是什么
- **人的价值重新锚定**：只做纠偏的人，成长路径是什么？留不住人，连纠偏的人都没有

### 问题四：遗漏了知识创造功能

组织不只是"分工→执行"的机器，还是**知识创造的场域**（野中郁次郎 SECI 模型）。人与人之间非正式的碰撞、争论、共创，是产生创新的关键机制。全换成 Agent，执行效率上去了，但创新能力可能会塌缩。

## 修正后的组织模型

```
┌─────────────────────────────────────────────┐
│  老板层：战略方向 + 资源分配决策               │
└──────────────────┬──────────────────────────┘
                   │ 双向反馈（不是单向拆解）
                   ↕
┌─────────────────────────────────────────────┐
│  关键人才层：感知 · 判断 · 纠偏 · 创新         │
│  （Agent 编排者 + 判断者 + 创新者）            │
└──────────────────┬──────────────────────────┘
                   │ 任务编排 + 质量审计
                   ↕
┌─────────────────────────────────────────────┐
│  Agent 层：高效执行结构化任务                   │
└─────────────────────────────────────────────┘
```

**核心判断**：中间层不是被压缩掉了，而是角色彻底变了。人数大幅减少，但能力要求大幅提升。"感知-判断-创新"这条人类独有价值链不可替代。
