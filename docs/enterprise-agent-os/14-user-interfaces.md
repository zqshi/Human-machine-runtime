# 14 — 用户触点层：不同角色的操作界面

## 核心原则

**这层决定产品能不能卖出去。** 底下再强，用户摸不到就是零。不同角色需要完全不同的界面。

---

## 老板看到的——战略驾驶舱

| 功能 | 说明 |
|------|------|
| 战略目标达成进度 | 不是甘特图，是**置信度仪表盘**（目标达成的概率而非进度百分比） |
| 关键风险信号 | 系统主动推送，不是要老板去翻 |
| 决策队列 | 需要老板决策的问题，精简到每天不超过 3 个 |

### 设计要点

老板的时间最贵。界面的核心指标是**老板每天在这个系统上花多少时间**——越少越好，但每一分钟都是高价值的。

---

## 关键人才看到的——判断工作台

| 功能 | 说明 |
|------|------|
| 实时信号流 | 按重要性排序，不是按时间 |
| 待判断队列 | 系统识别出需要人介入的决策点 |
| 纠偏操作台 | 做完判断后一键传播到下游所有受影响的 Agent 和任务 |
| Agent 集群状态 | 哪些在跑、哪些卡住了、哪些需要干预 |

### 设计要点

核心体验是"**问题来找人，而不是人去找问题**"。信号聚合和优先级排序的质量直接决定这个工作台的价值。

---

## HR 看到的——人机编制看板

| 功能 | 说明 |
|------|------|
| 能力覆盖地图 | 当前人 vs Agent 的能力覆盖范围可视化 |
| 能力缺口分析 | 哪些判断领域缺人，需要招聘或培养 |
| Agent 替代趋势 | 哪些人类任务正在被 Agent 接管，人力编制如何动态调整 |
| 人才效能分析 | 基于双轨考核数据，人才在判断质量维度的表现 |

### 设计要点

HR 的界面不是传统的 HR SaaS（考勤、薪酬、审批），而是**组织能力的实时体检报告**。
