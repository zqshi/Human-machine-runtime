# 04 — 竞品对比与品类定位

## 核心判断

Enterprise Agent OS 是一个**全新品类**，现有产品均不覆盖其完整能力范围。

## 现有品类对比

| 现有品类 | 代表产品 | 做了什么 | 缺什么 |
|---------|---------|---------|--------|
| 项目管理 | Jira / 飞书项目 / Asana | 人的任务管理 | 不理解 Agent 能力边界，任务粒度和结构不适配 Agent |
| Agent 框架 | CrewAI / AutoGen / LangGraph | Agent 技术编排 | 纯技术视角，不理解组织和战略，无组织级上下文 |
| BI / 数据平台 | Tableau / Metabase / 帆软 | 历史数据分析 | 只看历史，不做任务编排和判断路由 |
| ERP / OA | SAP / 钉钉 / 飞书 | 流程管理 | 流程固化，无法处理 Agent 的动态性 |
| Copilot 类 | GitHub Copilot / Microsoft 365 Copilot | 个人 AI 助手 | 个人视角，没有组织级编排能力 |
| 战略管理 | Cascade / Workboard | 战略目标追踪 | 只是 OKR 看板，无执行层闭环 |

## 品类定位

```
                    组织视角
                       ↑
                       │
    战略管理工具        │        Enterprise Agent OS
    (Cascade)          │        (目标位置)
                       │
  ──────────────────────┼──────────────────────→ Agent 能力
                       │
    项目管理工具        │        Agent 框架
    (Jira)             │        (CrewAI/LangGraph)
                       │
                    个体视角
```

Enterprise Agent OS 占据的是**右上象限**：同时具备组织视角和 Agent 编排能力。这是当前市场的空白地带。

## 潜在竞争风险

1. **飞书/钉钉等协作平台向右上扩展**：已有组织数据，如果加入 Agent 编排能力，威胁最大
2. **CrewAI/LangGraph 等向左上扩展**：技术底子好，但补组织理解很难
3. **大厂自研**：字节、阿里有数据优势和组织场景，但大厂通常做不好这类需要深度行业理解的产品

## 差异化壁垒

真正的壁垒不在技术，在于**对组织运转逻辑的理解深度**——这是绝大多数做 Agent 框架的技术团队不具备的。
