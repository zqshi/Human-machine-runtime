# 08 — 人机分工模型

## 分工的底层逻辑

不按"岗位"分，按**决策类型**分：

| 决策特征 | 归谁 | 原因 |
|---------|------|------|
| 有明确规则和标准答案 | Agent | 规则执行是 Agent 的绝对优势 |
| 数据充分、模式已知 | Agent | 模式匹配和数据处理效率碾压人 |
| 信息模糊、需要直觉判断 | 人 | 弱信号识别是人的核心能力 |
| 涉及利益博弈和政治 | 人 | Agent 不理解权力结构 |
| 需要创造性突破 | 人 | Agent 擅长组合已知，不擅长发明未知 |
| 有伦理 / 合规风险 | 人（最终决策） | 责任必须由人承担 |
| 高频重复但需要微调 | 人 + Agent 协作 | Agent 执行，人设参数和审核 |

## 动态分工机制

分工不是一次定死的，是**动态的**：

```
新任务出现
    ↓
系统评估：模糊度、风险等级、历史成功率
    ↓
┌──────────────────────────────────────────────┐
│ 高确定性 + 低风险 → Agent 自主执行             │
│ 高确定性 + 高风险 → Agent 执行 + 人审批        │
│ 低确定性 + 低风险 → Agent 尝试 + 人审核        │
│ 低确定性 + 高风险 → 人主导 + Agent 辅助        │
└──────────────────────────────────────────────┘
    ↓
执行完成后，更新该任务类型的确定性评分
    ↓
下次同类任务，分工可能发生变化
```

### 自学习特性

系统会**自动学习哪些任务可以越来越多地交给 Agent**，人的精力逐步聚焦到真正需要人的地方。随着数据积累，越来越多的 L1 任务会下沉到 L2，人的工作会越来越聚焦于高模糊度、高风险的判断。
