## 版本目标

打通当前最深的债——**声明层与执行层脱节**:`ResourceConfig` 声明完整(CPU/内存/GPU/PVC/model/budget)但 `ContainerOrchestratorProvisioner` 不传资源、`DockerWorkerRunner` 硬编码 `--memory 2g --cpus 1.0`、workspace 是 `/tmp` 临时目录无 PVC。同时建立 Agent 定义 CRD,把散落在 instance 表的声明收敛为版本化 spec。

## 版本验证标准

1. 配 `ResourceConfig.cpu=1000m` → `docker run` 参数含对应 `--cpus` 值(资源声明真生效)
2. 实例 workspace 跨 task 持久(per-instance 目录,实例删除才清理)
3. `agent_definitions` 表 CRUD 可用,spec 含 version/sandboxTemplate/resourceLimits/workspaceStrategy/boundTools/boundSkills/modelConfig

## 版本范围

**做什么**:
- Agent 定义 CRD:新建 `agent_definitions` 表,复用现有 ResourceConfig 结构
- 资源下沉:provisioner 传资源 + docker-runner 读 opts.resources(向后兼容缺省现值)
- PVC 落地:workspace per-instance 持久目录
- migration:agent_definitions 表 + instance 表加 agentDefinitionId/agentGeneration(§7.2.1 规则3+4)

**不做什么**(后续版本):
- reconcile 声明/运行解耦 → v1.5
- 工具/skill 自动组装 → v1.4
- 全链路 trace → v1.6
- eval 真实化 → v1.7

## 各任务实施要点

### T1 — Agent 定义 CRD

新建 `agent_definitions` 表,spec 字段(参照云原生平台设计文档 §4.v1.3):
- `version`(int,spec 世代,递增)
- `sandboxTemplate`(varchar,引用沙箱模板名,v1.4 才有模板池,本期先存名)
- `resourceLimits`(jsonb,引用 ResourceConfig 结构)
- `workspaceStrategy`(jsonb:`{type:'pvc'|'emptyDir', size}`)
- `boundTools`(jsonb:string[])
- `boundSkills`(jsonb:string[])
- `modelConfig`(jsonb)

domain 层定义 AgentDefinition + 校验(复用 validateResourceConfig)。repository 走 Drizzle 模式(参照 credential-repository.ts)。

### T2 — 资源下沉

`ContainerOrchestratorProvisioner.provision` 现只传 appKey/userID/name,补传 instance.resources.compute.cpu/memory(需确认 gateway 是否支持资源参数,不支持则 v1.4 沙箱模板统一处理)。向后兼容:resources 缺省时不传。

### T4 — docker-runner 读 resources + PVC

`docker-worker-runner.ts` 的 `buildArgs` 把硬编码 `--memory 2g --cpus 1.0` 改为读 `opts.resources`(WorkerRunOptions 加 resources 字段,缺省用 `2g/1.0` 保兼容)。`claude-agent-sdk-adapter.ts` 的 cwd 从 `${workspaceRoot}/${taskId}` 改 `${workspaceRoot}/${instanceId}`(per-instance 持久),task 级目录改为子目录。

## 对后续版本的约束

1. **ResourceConfig 必须被执行层消费**:v1.3 后,任何资源声明字段必须下沉到 provisioner/runner,禁止硬编码资源参数。
2. **workspace per-instance**:workspace 以 instanceId 为根,task 为子目录,保证实例级持久。

## 遗留(版本完成后补)

- **AgentDefinitionRepository 无 service/route**:v1.3 只落 schema+domain+repository,未建 application service + admin 路由(CRUD API)。v1.4 组装层或独立补。当前 CRD 可经 DB 直接操作。
- **provisioner 资源下沉未做**:gateway createInstance 契约不支持资源参数(上游固定),provisioner 不动。资源声明仅 docker-runner 层生效(claude-worker 沙箱)。上游 IM workspace 资源由 claw-farm 自管。
- **workspaceStrategy.size 声明未对接实际 PVC 配额**:v1.3 改 per-instance 目录(workspaceRoot/instanceId/taskId),但 storage.persistentVolumeSize 仍是声明未强制配额(无 filesystem quota)。配额强制留待真实 PVC 场景。
