## 版本目标

把被 `claude-agent-sdk` 主路径绕开的外部服务契约补回，让 claude-worker 在私有化（内网无 Anthropic 出口）场景可用。原 v1.2.0 范围过大，按 §11 拆为本版本（外部契约）+ v1.2.2（商业模式）。

## 交付内容摘要

- **T3 claude-worker 双路径**：路径A `ANTHROPIC_BASE_URL` 注入（config/adapter/WorkerRunOptions/docker-runner/bootstrap 四处 + 2 unit test），claude-worker 经企业代理转发，保留 SDK 完整能力；路径B LiteLLM+AgentExecutor 降级（已就绪）+ docker-compose litellm 编排 + 路径选择文档。
- **T1/T2**：契约对照表确认 `ANTHROPIC_API_KEY` 是唯一被绕开直连点；降级守卫核查发现现有架构已完整（调用方守卫模式），无需实质改动。
- **T5 + CI**：集成测试跨平台改进（`--network host` → bridge + `host.docker.internal:host-gateway` + fake server 监听 `0.0.0.0`，断言对齐 SDK 0.1.0 真实行为）；ci.yml 加 integration job。
- **配置 P0 修复（T11）**：.env.example 补 7 字段；.env.production.example + docker-compose 变量名漂移修复（CLAWHUB→MARKETPLACE 等 4 组）+ 删 CLAW_MANAGER 死配置；production-bootstrap 密码统一 HMR_SEED_ADMIN_PASSWORD。
- **credential-vault 后端管理面（T10）**：CredentialRepository + CredentialManagementService + routes/admin/credentials.ts（CRUD+lease，挂 auth，分页）；解锁 tool-management db 连接 getCredential 能力。
- **bootstrap.ts 拆分**：拆出 bootstrap/credentials.ts（804→795 行，脱离 800 红线区）。

## 关键技术决策表

| 决策 | 选择 | 理由 |
|------|------|------|
| claude-worker 私有化路径 | 双路径并存（A: BASE_URL 转发；B: LiteLLM 降级） | A 保留 SDK 完整能力但有国产模型兼容性风险；B 确定可用但放弃 Agent SDK 高级能力，环境自选 |
| SDK 是否读 ANTHROPIC_BASE_URL | 是（不改 worker.ts，SDK 自动读 env） | 集成测试实测：claude-worker 容器经 host.docker.internal 命中 fake server 10 次 + worker 输出 session_id/done 正常退出 |
| 集成测试网络模式 | bridge + host-gateway（非 --network host） | mac docker desktop 的 host 网络不能访问 mac loopback；bridge + host.docker.internal 跨平台 |
| gateway clients 降级守卫 | 不补码，核查确认现有架构已完整 | 配置判断是调用方职责（gateway-clients.ts 注释），强加 client 方法守卫违反 §4.3 |
| credential-vault 前端 | 延后至 backlog D4 | 后端 API 完整可用，前端缺失不阻断私有化部署 |
| 范围控制 | eval/knowledge-memory/tool 三缺口记 backlog | 严格 v1.2.1 外部契约范围，符合 §4.3 scope 纪律 |

## 对后续版本的约束

1. **gateway 契约不可再被绕开**：任何新外部服务调用必须走 `gateway/clients`，PR 审查强制检查。违反 → §12 标记 P1。
2. **外部契约 mock 必须进 CI**：集成测试套件是合入门槛。
3. **bootstrap 拆分延续**：新依赖组装优先提取到 bootstrap/ 子模块，保持 bootstrap.ts < 800 行。
