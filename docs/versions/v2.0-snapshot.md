## 关键技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| bake 同步 vs 异步 | 同步返回终态 | 低频管理操作+本地DB查询，同步比异步轮询简单且无 pending 卡死；消除 BakeResult `as` 类型欺骗 + 前端轮询缺失 |
| compiledTools 结构 | toolId+name+description+inputSchema | 兼容 T18b-A externalTools |
| compiledQuota | jsonb 快照 | 守 §1.1 domain 零外部依赖 |
| bake persona 来源 | 声明态 spec.persona | 非运行时 personaProvider 召回，固化可重放 |
| C14 发布即固化 | AgentCreateFlow createOrUpdate 后自动 bake | 失败不阻断可面板重试 |
| manifest 灰度 | 无 manifest 降级 assemble+getPersona | 存量未 bake Agent 不挂（不可跳过降级分支）|
| listByDefinition 分页 | limit 下推 DB 层 | §7.2.1#2 治本，消除假分页 |
| EAOS 接真LLM vs 诚实化 | AI 能力接真 LLM / 数据来源诚实化 | /decode+洞察接 LLM；涌现信号来自 dispatch 异常检测/编排路由来自真调度，诚实化不伪装不留 if/else 假智能 |

## 对后续版本的约束

1. **manifest 成为运行时真相源**：bake 普及后 assembly-provider/personaProvider 可收敛删除（单独版本），届时 harness 只读 manifest。
2. **sandbox 多后端路由固化**：SandboxRouter 模式确立，新增沙箱后端（Firecracker/gVisor）只加 executor + router 分支。
3. **Agent 行为不可变契约**：baked manifest 不可变，线上 Agent 行为可锁定/审计/回滚——是计费可信前置（计费基于固化配额才不漂移）。
4. **版本号纪律**：snapshot 不可改，版本号不重复用（v1.3 已归档占用，故升 v2.0）。
