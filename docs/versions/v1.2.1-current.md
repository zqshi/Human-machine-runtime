## 版本目标

**外部契约补强**: `claude-agent-sdk` 当前在 Docker 容器内直连 Anthropic API，绕开了 `gateway` context 对 LiteLLM/clawhub/portal/xspace/claw-farm 的契约封装，导致外部依赖不可替换、不可 mock、不可降级。本版本把主路径接回契约层。

## 版本验证标准

1. Agent 主路径（claude-agent-sdk 适配器）通过 `gateway/clients` 调用外部服务，docker 容器内不再硬编码外部 endpoint
2. 可在 `adapters` 层 mock 全部外部依赖跑通集成测试
3. gateway clients 补齐降级守卫与超时/重试策略（不配置时返回 `not configured` 而非抛错）

## 版本范围

**做什么:**
- `agent-core/sandbox/claude-agent-sdk-adapter.ts` 主路径接回 `gateway/clients`（去 sdk 直连）
- `gateway/clients` 补齐降级守卫与超时/重试
- 外部契约集成测试套件（mock LiteLLM 等）

**不做什么:**
- 商业模式（计费/套餐闭环）→ v1.2.2
- SSO 接入 → v1.3+（`system_configs` 已预留开关）
- `openclaw` 子系统治理 → 独立处置（不在 HMR 主路径）
- 数据层抽象（React Query/Suspense）→ 长期债务（见 `backlog.md`）

## 各任务实施要点

### T1 — gateway 契约缺口梳理

**产出**: 一份对照表，列出每个外部服务（LiteLLM/clawhub/portal/xspace/claw-farm）当前被 `claude-agent-sdk` 直接调用的点 vs `gateway/clients` 已封装的契约。

**方法**:
1. `grep` 项目内 `claude-agent-sdk` 的 import 与调用点
2. 重点核查 `claude-agent-sdk-adapter.ts` 内 docker worker 启动时传入的 env（是否含直连 endpoint）
3. 对比 `gateway/clients/{litellm,marketplace,profile-service,workspace-backend,container-orchestrator,cluster-instance}-client.ts` 的契约面
4. 未走契约层的 → 记录调用点 + 应接入的 client 名

**注意**: 这是诊断任务，不改代码。产出物作为 T2/T3 的输入。

### T3 — agent-core 接回契约 `[核心风险点]`

**风险**: `claude-agent-sdk` 在 docker 容器内运行，需要把外部 endpoint 配置通过 env 注入容器。替换不是简单改 import，而是改 docker worker 启动协议。

**回滚方案**: 保留 sdk 直连路径作为 feature flag 降级，灰度切换。

## 对后续版本的约束

1. **gateway 契约不可再被绕开**: 本版本后，任何新外部服务调用必须走 `gateway/clients`，PR 审查强制检查。违反 → §12 领域模型健康度审计标记 P1。
2. **外部契约 mock 必须进 CI**: T5 完成后，集成测试套件成为合入门槛。
