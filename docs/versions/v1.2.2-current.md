## 版本目标

1. **计费写侧链路补全**: 当前 `billing` context 是骨架（仅 recordEvent 累加 amount），缺 quota 扣减、对账任务、invoice 生成
2. **用量报表 API + 前端**: 管理后台能看到按租户/套餐维度的用量报表（API 调用次数、实例运行时长、Token 消耗）
3. **可观测性补强**: 商业模式上线必须有用量审计 + 异常告警，否则计费不可信

## 版本验证标准

1. 租户在 Pro 套餐下创建数字员工实例，超配额时返回 `QUOTA_EXCEEDED`（已有，端到端验证）
2. 管理后台 `/admin/billing` 能看到按租户/套餐维度的用量报表
3. billing 写侧链路完整：预占 → 实扣 → 对账（日终核对 quota 余量 vs analytics 累计 vs billing 累加）
4. 用量异常（超阈值）触发通知通道告警

## 版本范围

**做什么:**
- `quota-management` 与 `analytics` 数据流贯通（实时扣减 + 日终对账）
- billing 写侧补全：consume/deduct/reserve API、对账任务
- 用量报表 API（按租户/套餐/时段聚合）
- 前端用量报表展示
- 用量异常告警（阈值 + 通知通道）

**不做什么:**
- 真实支付网关对接 → v1.3+（本版本只做用量统计 + invoice 生成，不做扣款）
- 账单 PDF 导出 → v1.3+
- 多币种支持 → v1.3+（当前固定 USD）
