## 版本目标

投产审计（对照 `harness-knowledge-base` 生产级 Agent 上线检查表 15 项）结论：质量门禁全绿、核心链路完整，**具备内部灰度试运行条件**，但有 3 项阻断生产级全量上线（#1 拒答 / #7 执行审批 / #13 灰度）。同时 `AgentDefinition` CRD 是半成品（有 domain/repo/schema/组装消费，**无 CRUD API、实例未关联、前端创建流程脱节**）。本版本：补全 3 项阻断项 + 打通两条路径 + 治本 D8（openclaw 运行时解耦），使项目满足生产级全量上线标准。

## 版本验证标准

1. 创建/编辑 Agent 时能声明人设/模型/技能/知识/运行时/guardrails，落库 `agent_definitions` CRD，实例可关联
2. 数字员工实例路径（harness）+ openclaw 对话路径（useAgentChat）都注入 persona + guardrails，越界请求被拦截返回拒答
3. 高风险工具调用按 risk 等级 + 实例阈值走审批队列，admin approve/reject 后续执行
4. feature flag 能按租户 + 比例灰度（guardrails/approval 可 off/log/enforce 三态）
5. 管理后台可升级补充运行时/沙箱模板（治本 D8 后端侧）
6. 三重门禁全绿，现有 1455 测试不回归，useAgentChat IM 模式真请求浏览器验证

## 对后续版本的约束

1. **AgentDefinition spec 扩展兼容旧数据**：persona/boundKnowledge/runtime 有 default，旧实例未关联 agentDefinitionId 走默认（不阻断）
2. **guardrails/approval 默认 off**：enforce 三态（off/log/enforce），默认 off 不阻断现有行为，灰度逐步 enforce
3. **AgentRuntimePort 抽象后**：新增运行时必须实现该 port（治本 D8 的硬约束）
4. **CRD 声明态是 Agent 创建唯一入口**：禁止绕开 AgentDefinition 直接配置运行时（PR 审查强制）
