## 关键技术决策表

| 决策 | 选择 | 理由 |
|------|------|------|
| 组装层架构 | 方案 B(IAssemblyProvider 与 RagContextProvider 并存) | 语义正交,避免 §12 信号 2 |
| boundTools 语义 | 存 definitionId | 可定位 source/凭证,支持未来动态工具 |
| trace context 传递 | 显式传参(harness 局部) | 不依赖事件总线(fire-and-forget) |

| 跨聚合依赖 | port 接口(D2/v1.4/v1.6/v1.7 全用) | 守 §1.3,agent-core domain 不依赖其他 context |
| 注入时机 | setter 延后注入 | 依赖 repo 晚于 harness 实例化 |

## 共同设计模式(贯穿 4 版)

- **harness.dispatchTask 汇聚点**:RAG/assembly/trace/eval 都在此注入(D2 开先例),单一注入点
- **port 接口 + bootstrap 适配**:每个跨聚合能力(domain 定义 port,bootstrap 适配实现)
- **复用雏形不造轮子**:ResourceConfig/distributed_traces/evaluator-engine/ToolManagementService 演进不新建
- **容错不阻断主链路**:RAG/assembly/trace 失败都不阻断任务执行(try/catch + appEventBus)

## 遗留问题

- **v1.8 reconcile(原 v1.5)**:声明/运行双态分离 + spec-diff 调和(唯一未做)
- **各版端到端实测**:v1.3 CRD/v1.4 组装/v1.6 trace/v1.7 eval 均单测覆盖,真实 DB+LiteLLM+工具预置的端到端未跑
- **CRD admin service/route**:AgentDefinitionRepository 已接入 bootstrap(assembly 用),无 CRUD API
- **worker 内部 trace 埋点**:跨进程 NDJSON span 上报,协议已预留
- **孤立 LLM span 接根**:chat.ts/ai-assistant.ts/trace-sync-job.ts 独立入口,待收敛
- **eval 真实工具预置**:case expectedTools 需 ToolDefinition 存在(数据准备)

## 对后续版本的约束

1. **harness.dispatchTask 是汇聚点**:新能力优先在此注入,不散落
2. **声明优先于执行**:新功能先建模 spec(存 DB),再接执行层
3. **port 接口守 §1.3**:跨聚合能力一律 port + bootstrap 适配
4. **复用雏形**:ResourceConfig/distributed_traces/evaluator-engine 已达生产级,演进不新建
5. **migration 随 schema**:新表/加列同步 .ts migration(§7.2.1 规则3)
