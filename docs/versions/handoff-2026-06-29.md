## 三、本会话教训
- **接真 LLM vs 诚实化**：本就是 AI 能力的（战略解码/洞察生成）接真 LLM；本不该 LLM 生成、应来自数据的（涌现信号来自 dispatch 异常检测/编排路由来自真调度）诚实化不伪装，不留 if/else 假智能。
- **全量 vitest 必须在 server 目录跑**（`cd server && npx vitest run`）；根目录会用到 monorepo v4.1.9 导致 `describe is not defined` 假失败（10 文件 0 test）。eslint 用 `cd server && npx eslint src/`。
- 服务用 tsx watch 热重载，改源码自动重载，无需手动重启即可实测。
