## 决策表

| 决策 | 选择 | 理由 |
|------|------|------|
| 拒答机制 | systemPrompt 软约束 + guardrails 硬约束 + LLM 复核 | 三层防御，复刻 RagContextProvider LLM 判断 |
| 审批力度 | riskLevel + 实例 approvalPolicy 阈值 + ApprovalStore | 复用 marketplace IApprovalStore 模式 |
| 灰度开关 | enforce 三态（off/log/enforce） | 复用 enforceModelGrants |
| T9 persona 链路 | 完整治本：instance→agentDefinitionId→AgentDefinition.persona | 后端 instance 已透传 agentDefinitionId（T7+repo），前端 instanceApi.get 已就绪，无需改后端 |
| T9 openclaw guardrail | 前端拦截为主 | openclaw chat 经 LiteLLM 不经 harness，后端兜底待 openclaw chat route 接入 |
| T10 feature flag HTTP | 补 routes/admin/feature-flags | T5 仅 service+middleware 无 route，flag UI 需 HTTP 端点 |

## 遗留

1. **T9 流式真请求待浏览器实测**：useAgentChat 重构触及 openclaw/weKnora 流式主路径，tsc/vitest 测不出，需 IM 模式浏览器实测回归（同 D10 风险类）
2. **openclaw 路径后端 guardrail 兜底**：openclaw chat route（/api/openclaw/chat）经 LiteLLM 不经 harness，T3 guardrail 仅实例路径 harness 生效；openclaw 路径 guardrail 仅前端拦截，后端兜底待 openclaw chat route 接入 guardrail
3. **persona 数据源依赖 activeInstanceId**：前端 instance→agentDefinitionId 链路依赖 IM 对话上下文有 activeInstanceId；无 active instance 时无 persona 注入（兼容，无 guardrail）

## 约束（对后续版本）

1. AgentDefinition spec 扩展兼容旧数据（persona/boundKnowledge/runtime 有 default，旧实例未关联 agentDefinitionId 走默认不阻断）
2. guardrails/approval 默认 off（enforce 三态），灰度逐步 enforce，不阻断现有行为
3. AgentRuntimePort 抽象后：新增运行时必须实现该 port（D8 治本硬约束，PR 审查强制）
4. CRD 声明态是 Agent 创建唯一入口（禁止绕开 AgentDefinition 直接配置运行时）
5. T9 遗留（见上）：openclaw guardrail 后端兜底 + 流式实测，下版本或浏览器验证后关闭
